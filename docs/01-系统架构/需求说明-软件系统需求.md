# 智慧酒店物联网控制系统 - 软件系统需求

## 📋 文档概述

### 文档说明
本文档描述智慧酒店物联网控制系统的软件系统需求，包括云端服务器、Web管理后台、手机端H5/小程序、前端界面等各个软件模块的功能需求、技术实现和接口设计。

### 适用范围
- 云端服务器开发
- Web管理后台开发（Vue.js 3）
- 手机端H5/小程序开发（住客/员工端）
- 嵌入式固件开发（前台管理端/楼控/客房端）
- 前端Web界面开发

### 文档版本
- **v6.0.0** - 2026年4月
  - **新增客房端语音通话功能**：与前台实时双向语音通信
  - **新增 WebRTC/MQTT 音频传输适配需求**
  - **新增前端语音通话 UI 组件选型**
  - v5.0.0 基础上重构为智慧酒店三层架构
  - 新增预订系统、支付接口、会员机制、优惠券机制
  - 移除桌面应用，统一使用Vue Web管理后台
  - 保留AI管家系统，前端语音识别（APP端ASR），统一后端处理文字指令

### 1.2 技术栈

**前端技术栈（统一Vue框架，多角色视图）**：
- Web前端：Vue.js 3 + Ant Design Vue + ECharts
- 状态管理：Pinia
- HTTP客户端：Axios
- WebSocket客户端：Socket.io-client
- 路由管理：Vue Router 4
- 组件库：Ant Design Vue (UI组件) + Chart.js (图表)
- 多角色视图架构：
  - **管理员视图**（Web管理后台）：入住/退房管理、房卡管理、客房状态监控、预订管理、数据分析报表
  - **住客视图**（手机端H5/小程序）：客房服务请求、设备控制、酒店信息、个人中心
  - **员工视图**（服务员/维修工移动端）：客房状态查看、服务记录

**后端技术栈**：
  - 后端框架：Node.js + Express
  - 数据库：MySQL（主数据库）+ Redis（缓存）
  - 消息队列：MQTT Broker（Mosquitto）
  - 实时通信：WebSocket（Socket.io）

**部署架构**：
  - 前端部署：Nginx 反向代理，静态资源托管
  - 后端部署：Node.js 服务（PM2 进程管理）
  - 数据库部署：MySQL 主从复制 + Redis 集群


**后端技术栈**：
- 后端框架：Node.js + Express
- 数据库：MySQL（主数据库）+ Redis（缓存）
- 消息队列：MQTT Broker（Mosquitto）
- 实时通信：WebSocket（Socket.io）

## ☁️ 云端服务器需求

### 2.1 数据管理

#### 2.1.1 数据存储

**功能描述**：存储所有酒店数据、用户数据、系统日志

**存储内容**：
- **酒店信息**：酒店基本信息、房间信息、设施信息、周边推荐
- **房间信息**：房号、房型、价格、状态（空闲/入住/维修/SOS）
- **住客信息**：姓名、身份证号、房号、入住时间、退房时间
- **房卡信息**：房卡号、房号、有效期、权限等级
- **预订信息**：预订号、房号、入住日期、退房日期、状态、支付状态
- **支付信息**：支付单号、订单ID、金额、支付方式、交易ID、状态
- **会员信息**：会员ID、姓名、手机号、会员等级、积分、有效期
- **优惠券信息**：优惠券码、优惠券名称、优惠类型、折扣值、有效期、使用状态
- **设备信息**：设备ID、设备类型、设备状态、固件版本
- **传感器数据**：温度、湿度、空气质量、光照等
- **控制指令**：开灯、关灯、开门、空调控制等
- **送物订单**：订单号、房号、物品类别、物品名称、数量、备注、状态、时间
- **报修工单**：工单号、房号、故障类型、故障描述、照片、优先级、状态、处理人、时间
- **服务评价**：评价ID、房号、订单/工单ID、评分（1-5星）、评价内容、时间
- **用户信息**：用户名、密码、角色（管理员/服务员/维修工/住客）、权限
- **系统日志**：操作日志、错误日志、报警日志、入住/退房日志、支付日志、会员日志

**存储方式**：MySQL（主数据库）+ Redis（缓存）

**数据备份**：每日自动备份，保留90天

#### 2.1.2 数据查询

**功能描述**：提供数据查询接口

**查询功能**：
- 实时数据查询（房间状态、设备状态）
- 历史数据查询（入住记录、订单记录、工单记录）
- 统计数据查询（入住率、订单量、工单量、满意度）
- 设备状态查询（在线/离线、健康状态）
- 送物订单查询（按房号/订单号/状态查询）
- 报修工单查询（按房号/工单号/优先级/状态查询）
- 预订订单查询（按预订号/房号/日期查询）
- 支付记录查询（按支付单号/订单号/状态查询）
- 会员信息查询（按会员ID/手机号查询）
- 优惠券查询（按优惠券码/会员查询）

**查询方式**：RESTful API

**查询限制**：单次查询最多1000条记录

#### 2.1.3 数据分析

**功能描述**：对采集的数据进行分析

**分析功能**：
- 环境数据趋势分析（温度、湿度、空气质量）
- 设备使用频率统计（灯光、空调、窗帘使用频率）
- 异常数据检测（设备离线、传感器异常）
- 能耗分析（各房间能耗统计）
- 酒店运营分析（入住率、订单量、工单量、满意度）
- 送物订单分析（热门物品、配送时效）
- 报修工单分析（故障类型分布、处理时效）
- 预订分析（预订量、预订率、预订取消率、平均预订周期）
- 支付分析（支付方式分布、支付成功率、平均支付金额）
- 会员分析（会员数量、会员等级分布、会员活跃度、会员复购率）
- 优惠券分析（优惠券发放量、使用率、优惠券核销金额）

**分析结果**：生成报表和图表

### 2.2 设备管理

#### 2.2.1 设备注册

**功能描述**：设备启动时自动注册到后端

**注册信息**：
- 设备ID（唯一标识）
- 设备类型（前台管理端/楼控/客房端）
- 设备名称
- 设备能力（支持的传感器和执行器）
- 固件版本
- 注册时间

**注册时机**：设备启动时自动注册

**注册方式**：MQTT或HTTP

#### 2.2.2 设备认证

**功能描述**：验证设备身份，防止未授权设备接入

**认证方式**：设备ID + 设备密钥

**认证流程**：
1. 设备发送认证请求（设备ID + 设备密钥）
2. 后端验证设备身份
3. 验证通过，返回认证Token
4. 设备使用Token进行后续通信

**Token有效期**：24小时

#### 2.2.3 设备状态监控

**功能描述**：实时监控设备状态

**监控内容**：
- 在线/离线状态
- 设备健康状态
- 设备运行时间
- 设备错误信息

**监控方式**：心跳检测（30秒）

**离线报警**：设备离线超过5分钟，发送报警通知

#### 2.2.4 设备控制

**功能描述**：向设备发送控制指令

**控制方式**：MQTT消息

**控制指令**：
- 继电器控制（开/关）
- LED控制（颜色/亮度）
- 电控锁控制（开锁/上锁）
- 安防模式控制（布防/撤防）
- 空调控制（开关、温度、模式）
- 窗帘控制（开/关/暂停）
- 灯光控制（开关、亮度、颜色）

**控制响应**：设备返回执行结果

### 2.3 指令调度

#### 2.3.1 指令下发

**功能描述**：接收前端指令并下发给对应设备

**指令来源**：
- 前端用户操作（前台电脑/手机端）
- 定时任务
- 场景模式
- 语音指令

**指令处理**：
- 指令解析与路由
- 指令优先级管理
- 指令执行结果反馈
- 超时重试机制（3次）
- 指令日志记录

#### 2.3.2 指令路由

**功能描述**：根据设备类型和ID路由指令

**路由规则**：
- 前台管理端指令 → 前台管理端设备
- 楼控指令 → 楼控设备
- 客房端指令 → 客房端设备

**路由方式**：MQTT Topic路由

### 2.4 MQTT服务

#### 2.4.1 消息路由

**功能描述**：路由设备之间的消息

**MQTT Broker**：Mosquitto

**Topic设计**：
- 设备上行：`device/{device_id}/data`
- 设备下行：`device/{device_id}/command`
- 设备状态：`device/{device_id}/status`
- 广播消息：`broadcast/{message_type}`

**QoS等级**：QoS 1（至少一次）

**保留消息**：支持

#### 2.4.2 消息处理

**功能描述**：处理设备发送的MQTT消息

**处理流程**：
1. 接收MQTT消息
2. 解析消息内容
3. 验证消息格式
4. 存储到数据库
5. 转发到WebSocket客户端

**消息格式**：JSON

### 2.5 WebSocket服务

#### 2.5.1 实时通信

**功能描述**：提供实时通信能力

**WebSocket库**：Socket.io

**通信内容**：
- 传感器数据实时推送
- 设备状态实时推送
- 报警信息实时推送
- 控制指令实时推送
- 送物订单状态推送
- 报修工单状态推送

**推送频率**：
- 传感器数据：1-10秒
- 报警信息：实时
- 送物订单：实时
- 报修工单：实时

#### 2.5.2 客户端连接

**功能描述**：管理客户端WebSocket连接

**连接管理**：
- 客户端认证
- 连接状态管理
- 心跳检测
- 异常断开重连

**最大连接数**：100个并发连接

### 2.6 📞 语音通话服务（v6.0新增）

#### 2.6.1 功能概述
**功能描述**：提供客房端与前台管理端之间的实时双向语音通信能力

**业务场景**：
- **住客发起**：住客按客房终端"呼叫前台键" → 发起语音通话请求
- **前台发起**：前台主动呼叫指定房间（退房提醒、紧急通知、服务回访等）
- 前台Web界面收到来电通知 → 接听/拒接/转接
- 通话中双方实时语音交互（咨询、报修、送物请求等）
- SOS求助时自动升级为紧急语音通道

#### 2.6.2 技术方案选型

| 方案 | 架构 | 延迟 | 带宽 | 复杂度 | 推荐场景 |
|------|------|------|------|--------|----------|
| **A: WebRTC + Socket.io信令**（推荐） | P2P直连+STUN/TURN | **≤150ms** | 自适应(Opus) | 中等 | 生产环境，支持NAT穿透 |
| B: MQTT + Opus编码 | 中心转发 | ~200-300ms | ~32kbps | 低 | 纯局域网，复用MQTT基础设施 |
| C: WebSocket + PCM原始音频 | 中心转发 | ~150ms | ~256kbps | 最低 | 快速原型/MVP验证 |

**推荐方案 A（WebRTC）理由**：
1. **低延迟**：P2P直连避免服务器中转瓶颈
2. **自适应码率**：Opus 编码自动适应网络状况（8kbps~128kbps）
3. **回声消除(AEC)内置**：浏览器原生支持，无需额外开发
4. **NAT穿透**：STUN/TURN 解决酒店内网/公网互通问题
5. **前端生态成熟**：Vue 组件库丰富（vue-webrtc、simple-peer）

### 5. WebSocket实时通信

```javascript
// utils/websocket.js
import io from 'socket.io-client'

class WebSocketClient {
  constructor() {
    this.socket = null
    this.connected = false
  }

  connect() {
    this.socket = io(process.env.VUE_APP_WS_URL, {
      transports: ['websocket'],
      reconnection: true,
      reconnectionDelay: 1000,
      reconnectionAttempts: 5
    })

    this.socket.on('connect', () => {
      this.connected = true
      console.log('WebSocket connected')
    })

    this.socket.on('disconnect', () => {
      this.connected = false
      console.log('WebSocket disconnected')
    })

    this.socket.on('error', (error) => {
      console.error('WebSocket error:', error)
    })

    this.socket.on('message', (data) => {
      this.handleMessage(data)
    })
  }

  disconnect() {
    if (this.socket) {
      this.socket.disconnect()
      this.socket = null
      this.connected = false
    }
  }

  sendMessage(type, data) {
    if (this.connected && this.socket) {
      this.socket.emit('message', { type, data })
    }
  }

  handleMessage(data) {
    // 根据消息类型分发到不同的处理函数
    switch (data.type) {
      case 'room_status_update':
        this.handleRoomStatusUpdate(data.payload)
        break
      case 'work_order_update':
        this.handleWorkOrderUpdate(data.payload)
        break
      case 'notification':
        this.handleNotification(data.payload)
        break
      default:
        console.log('Unknown message type:', data.type)
    }
  }

  handleRoomStatusUpdate(payload) {
    // 更新客房状态
    console.log('Room status updated:', payload)
  }

  handleWorkOrderUpdate(payload) {
    // 更新工单状态
    console.log('Work order updated:', payload)
  }

  handleNotification(payload) {
    // 显示通知
    console.log('Notification:', payload)
  }
}

export default new WebSocketClient()
```

### 2. 常见问题

**Q: 智谱 GLM-4 免费吗？**  
A: 是的，GLM-4-Flash 完全免费，无限制。

**Q: ESP32 可以离线运行吗？**  
A: 可以，但需要在 ESP32 上运行轻量级模型（如 ESP-SR），性能会受限。

**Q: 如何降低延迟？**  
A: 使用阿里云 ECS，延迟可控制在 1-2秒内。

**Q: 小组成员如何协作开发？**  
A: 使用 GitHub 远程仓库，每人本地开发，提交代码后云服务器自动部署。

**Q: 手机端APP如何连接后端？**  
A: 手机端通过4G/WiFi连接后端服务器，使用WebSocket实现实时通信。

**Q: 前台电脑端如何连接硬件？**  
A: 前台电脑端通过USB串口连接前台管理端硬件，使用自定义JSON协议通信。

**Q: 移动端语音识别在哪里进行？**  
A: 移动端语音识别在前端进行，使用浏览器Web Speech API或第三方ASR服务，识别后的文字发送给后端AI管家处理。
