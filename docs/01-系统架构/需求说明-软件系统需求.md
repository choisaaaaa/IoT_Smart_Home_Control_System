## ☁️ 云端服务器需求

### 1. 系统架构

#### 1.1 架构设计
云服务器采用**前后端分离架构**：
- **前端服务器**：负责用户界面展示
- **后端服务器**：负责业务逻辑处理、设备管理、指令下发等核心业务

#### 1.2 技术栈

**前端技术栈**：
- Web前端：Vue.js 3 + Ant Design Vue + ECharts
- 移动端：Flutter App
- 状态管理：Pinia
- HTTP客户端：Axios
- WebSocket客户端：Socket.io-client

**后端技术栈**：
- 后端框架：Node.js + Express
- 数据库：MySQL（主数据库）+ Redis（缓存）
- 消息队列：MQTT Broker（Mosquitto）
- 实时通信：WebSocket（Socket.io）
- AI服务：语音识别API（阿里云）、语音合成API（讯飞/阿里）

### 2. 后端功能需求

#### 2.1 数据管理

##### 2.1.1 数据存储
- **功能描述**：存储所有设备数据、用户数据、系统日志
- **存储内容**：
  - 设备信息（设备ID、设备类型、设备状态）
  - 传感器数据（温度、湿度、空气质量、光照等）
  - 控制指令（开灯、关灯、开门等）
  - 用户信息（用户名、密码、权限）
  - 系统日志（操作日志、错误日志）
- **存储方式**：MySQL（主数据库）+ Redis（缓存）
- **数据备份**：每日自动备份，保留30天

##### 2.1.2 数据查询
- **功能描述**：提供数据查询接口
- **查询功能**：
  - 实时数据查询
  - 历史数据查询
  - 统计数据查询
  - 设备状态查询
- **查询方式**：RESTful API
- **查询限制**：单次查询最多1000条记录

##### 2.1.3 数据分析
- **功能描述**：对采集的数据进行分析
- **分析功能**：
  - 环境数据趋势分析
  - 设备使用频率统计
  - 异常数据检测
  - 能耗分析
- **分析结果**：生成报表和图表

#### 2.2 设备管理

##### 2.2.1 设备注册
- **功能描述**：设备启动时自动注册到后端
- **注册信息**：
  - 设备ID（唯一标识）
  - 设备类型（主控/分控1/分控2）
  - 设备名称
  - 设备能力（支持的传感器和执行器）
  - 固件版本
  - 注册时间
- **注册时机**：设备启动时自动注册
- **注册方式**：MQTT或HTTP

##### 2.2.2 设备认证
- **功能描述**：验证设备身份，防止未授权设备接入
- **认证方式**：设备ID + 设备密钥
- **认证流程**：
  1. 设备发送认证请求（设备ID + 设备密钥）
  2. 后端验证设备身份
  3. 验证通过，返回认证Token
  4. 设备使用Token进行后续通信
- **Token有效期**：24小时

##### 2.2.3 设备状态监控
- **功能描述**：实时监控设备状态
- **监控内容**：
  - 在线/离线状态
  - 设备健康状态
  - 设备运行时间
  - 设备错误信息
- **监控方式**：心跳检测（30秒）
- **离线报警**：设备离线超过5分钟，发送报警通知

##### 2.2.4 设备控制
- **功能描述**：向设备发送控制指令
- **控制方式**：MQTT消息
- **控制指令**：
  - 继电器控制（开/关）
  - LED控制（颜色/亮度）
  - 电控锁控制（开锁/上锁）
  - 安防模式控制（布防/撤防）
- **控制响应**：设备返回执行结果

#### 2.3 指令调度

##### 2.3.1 指令下发
- **功能描述**：接收前端指令并下发给对应设备
- **功能要求**：
  - 指令解析与路由
  - 指令优先级管理
  - 指令执行结果反馈
  - 超时重试机制（3次）
  - 指令日志记录
- **指令来源**：
  - 前端用户操作
  - 定时任务
  - 场景模式
  - 语音指令

##### 2.3.2 指令路由
- **功能描述**：根据设备类型和ID路由指令
- **路由规则**：
  - 主控指令 → 主控设备
  - 分控1指令 → 分控1设备
  - 分控2指令 → 分控2设备
- **路由方式**：MQTT Topic路由

#### 2.4 MQTT服务

##### 2.4.1 消息路由
- **功能描述**：路由设备之间的消息
- **MQTT Broker**：Mosquitto
- **Topic设计**：
  - 设备上行：`device/{device_id}/data`
  - 设备下行：`device/{device_id}/command`
  - 设备状态：`device/{device_id}/status`
  - 广播消息：`broadcast/{message_type}`
- **QoS等级**：QoS 1（至少一次）
- **保留消息**：支持

##### 2.4.2 消息处理
- **功能描述**：处理设备发送的MQTT消息
- **处理流程**：
  1. 接收MQTT消息
  2. 解析消息内容
  3. 验证消息格式
  4. 存储到数据库
  5. 转发到WebSocket客户端
- **消息格式**：JSON

#### 2.5 WebSocket服务

##### 2.5.1 实时通信
- **功能描述**：提供实时通信能力
- **WebSocket库**：Socket.io
- **通信内容**：
  - 传感器数据实时推送
  - 设备状态实时推送
  - 报警信息实时推送
  - 控制指令实时推送
- **推送频率**：传感器数据1-10秒，报警信息实时

##### 2.5.2 客户端连接
- **功能描述**：管理客户端WebSocket连接
- **连接管理**：
  - 客户端认证
  - 连接状态管理
  - 心跳检测
  - 异常断开重连
- **最大连接数**：100个并发连接

#### 2.6 API服务

##### 2.6.1 RESTful API
- **功能描述**：提供RESTful API接口
- **API框架**：Express.js
- **API版本**：v1
- **API基路径**：`/api/v1`

##### 2.6.2 API接口列表

**设备管理接口**
```
GET    /api/v1/devices              # 获取设备列表
GET    /api/v1/devices/:id          # 获取设备详情
POST   /api/v1/devices/:id/control  # 控制设备
DELETE /api/v1/devices/:id          # 删除设备
```

**数据查询接口**
```
GET    /api/v1/sensors/data         # 获取传感器数据
GET    /api/v1/sensors/data/latest  # 获取最新数据
GET    /api/v1/sensors/data/history # 获取历史数据
```

**用户管理接口**
```
POST   /api/v1/auth/login           # 用户登录
POST   /api/v1/auth/logout          # 用户登出
GET    /api/v1/users/profile        # 获取用户信息
PUT    /api/v1/users/profile        # 更新用户信息
```

**安防管理接口**
```
GET    /api/v1/security/status      # 获取安防状态
POST   /api/v1/security/arm         # 启动布防
POST   /api/v1/security/disarm      # 解除布防
GET    /api/v1/security/events      # 获取安防事件
```

##### 2.6.3 API认证
- **认证方式**：JWT（JSON Web Token）
- **Token有效期**：7天
- **认证流程**：
  1. 用户登录，获取Token
  2. 请求时在Header中携带Token
  3. 服务端验证Token
  4. 验证通过，返回数据

##### 2.7.4 智能体编排

### 4. 数据库设计

#### 4.1 数据库表结构

##### 4.1.1 设备表（devices）
```sql
CREATE TABLE devices (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id VARCHAR(50) UNIQUE NOT NULL,
    device_type VARCHAR(20) NOT NULL,
    device_name VARCHAR(50) NOT NULL,
    device_key VARCHAR(50) NOT NULL,
    device_status VARCHAR(20) DEFAULT 'offline',
    firmware_version VARCHAR(20),
    last_seen DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_device_id (device_id),
    INDEX idx_device_status (device_status)
);
```

##### 4.1.2 传感器数据表（sensor_data）
```sql
CREATE TABLE sensor_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id VARCHAR(50) NOT NULL,
    sensor_type VARCHAR(20) NOT NULL,
    sensor_value VARCHAR(50) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_device_id (device_id),
    INDEX idx_sensor_type (sensor_type),
    INDEX idx_created_at (created_at)
);
```

##### 4.1.3 控制指令表（control_commands）
```sql
CREATE TABLE control_commands (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id VARCHAR(50) NOT NULL,
    command_type VARCHAR(20) NOT NULL,
    command_value VARCHAR(50) NOT NULL,
    command_status VARCHAR(20) DEFAULT 'pending',
    created_by VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    executed_at DATETIME,
    INDEX idx_device_id (device_id),
    INDEX idx_created_at (created_at)
);
```

##### 4.1.4 用户表（users）
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    role VARCHAR(20) DEFAULT 'user',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

##### 4.1.5 安防事件表（security_events）
```sql
CREATE TABLE security_events (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id VARCHAR(50) NOT NULL,
    event_type VARCHAR(20) NOT NULL,
    event_data TEXT,
    event_level VARCHAR(20) DEFAULT 'info',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_device_id (device_id),
    INDEX idx_created_at (created_at)
);
```

### 4. 功能需求

#### 4.1 通信功能

##### 4.1.1 WiFi连接
- **功能描述**：连接WiFi网络
- **连接方式**：
  - 配网模式：扫描WiFi网络，用户选择并输入密码
  - 自动连接：使用已保存的WiFi信息
- **连接流程**：
  1. 扫描WiFi网络
  2. 选择目标WiFi
  3. 输入WiFi密码
  4. 尝试连接
  5. 连接成功，保存WiFi信息
  6. 连接失败，提示用户重新输入
- **自动重连**：WiFi断开后自动重连
- **重连间隔**：10秒

##### 4.1.2 MQTT通信
- **功能描述**：与后端服务器进行MQTT通信
- **MQTT Broker**：后端服务器
- **Topic设计**：
  - 上行Topic：`device/{device_id}/data`
  - 下行Topic：`device/{device_id}/command`
  - 状态Topic：`device/{device_id}/status`
- **QoS等级**：QoS 1
- **心跳间隔**：30秒

##### 4.1.3 HTTP通信

### 2. 常见问题

**Q: 阿里云 ASR/TTS 免费额度是多少？**  
A: 阿里云 Paraformer 流式语音识别提供 2个月免费（100小时/日），语音合成提供 200万字符/月免费。

**Q: 智谱 GLM-4 免费吗？**  
A: 是的，GLM-4-Flash 完全免费，无限制。

**Q: ESP32 可以离线运行吗？**  
A: 可以，但需要在 ESP32 上运行轻量级模型（如 ESP-SR），性能会受限。

**Q: 如何降低延迟？**  
A: 使用阿里云 ECS + 阿里云 API，延迟可控制在 1-2秒内。

**Q: 小组成员如何协作开发？**  
A: 使用 GitHub 远程仓库，每人本地开发，提交代码后云服务器自动部署。
