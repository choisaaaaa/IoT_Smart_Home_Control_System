# 智慧酒店物联网控制系统 - 整体需求

## 📋 项目概述

### 项目名称
**"智联酒店"智慧酒店物联网控制系统**

### 项目类型
物联网应用 - 智慧酒店小类

### 项目目标
开发一个低成本、高完成度的智慧酒店物联网控制系统，采用模块化、可复用的硬件设计，支持多房间批量组网和云端联控，实现客房智能化、运营管理数字化、能源管理精细化的智慧酒店整体解决方案。

### 核心特点
- **模块化设计**：硬件可复用，支持灵活配置
- **低成本**：单间客房平均成本¥90-¥145
- **高完成度**：20天内完成完整可演示系统
- **系统完整**：包含硬件、软件、云端、移动端全栈
- **易于实现**：使用成熟技术栈，学习曲线平缓
- **批量组网**：支持100+房间同时组网
- **云端联控**：支持前台集中控制、分区控制
- **酒店场景**：针对酒店行业痛点，如无接触入住、节能管理、安防监控

## 🎯 设备角色定义

### 1. 前台管理端（酒店级协调 + USB外设模式）

#### 功能定位
前台管理端是智慧酒店三层架构的前台核心设备，部署于酒店前台，负责入住退房办理、房卡管理、客房状态可视化、SOS报警、消防联动、公安对接等功能。

#### 核心职责
- **物理I/O交互**：RFID读卡、按键操作、蜂鸣器反馈、RGB灯带状态墙
- **本地报警**：SOS一键报警、蜂鸣器警报音效
- **外部接口**：RS485消防主机、UART公安终端
- **数据上报**：通过USB串口将数据上报前台电脑软件
- **状态可视化**：RGB灯带实时显示各客房状态

#### 硬件组成
- **主控芯片**：ESP32-S2-MINI / STM32F103C8T6
- **USB接口**：USB Type-C（供电+数据）
- **RFID模块**：RC522，13.56MHz（房卡读写）
- **RGB灯带**：WS2812B×8颗（客房状态可视化墙）
- **按键模块**：4键独立按键 + SOS报警键
- **蜂鸣器**：有源蜂鸣器，5V（操作提示音、报警音效）
- **RS485模块**：MAX3485（对接消防主机）
- **UART扩展口**：4Pin接插端子（预留身份证读卡器/公安终端）
- **电源**：USB取电（5V）

#### 通信方式
- **USB串口通信**：与前台电脑通信（自定义JSON协议）
- **无WiFi模块**：依赖前台电脑的网络连接

#### 成本预算
- **经济型**：¥84/台（USB外设模式，移除冗余硬件）

#### 职责分工
- **🖥️ 前台电脑**：GUI界面、AI语音助理、MQTT通信、数据存储、业务逻辑
- **🔌 本设备**：物理I/O组件（RFID/按键/蜂鸣器/灯带/RS485/UART）

---

### 2. 楼控（环境监测与服务支持）

#### 功能定位
楼控是智慧酒店三层架构的环境监测中心，部署于每层楼的公共区域，负责环境数据采集、数据上报、辅助设备控制等功能。

#### 核心职责
- **环境监测**：温湿度、空气质量、光照强度实时监测
- **数据采集**：1-10秒采集频率，误差 < 5%
- **数据上报**：MQTT协议上报云端
- **状态指示**：RGB LED显示工作状态（正常/异常/采集中）
- **辅助控制**：2路继电器控制（灯光/插座）

#### 硬件组成
- **主控芯片**：ESP32-WROOM-32（双核240MHz，WiFi+蓝牙）
- **传感器**：
  - DHT11（温湿度）
  - MQ-135（空气质量）
  - 光敏电阻（光照强度）
- **执行器**：
  - 继电器模块（2路，10A/250VAC）
  - RGB LED（WS2812B×5颗，状态指示）
- **电源**：DC 5V/1A，Micro USB

#### 通信方式
- **MQTT**：与云端服务器通信
- **HTTP**：数据上报

#### 成本预算
- **标准型**：¥105/台

---

### 3. 客房端（客房设备控制与状态反馈）

#### 功能定位
客房端是智慧酒店三层架构的客房控制中心，部署于每个客房内，负责客房设备控制、状态反馈、本地控制、远程控制等功能。

#### 核心职责
- **设备控制**：灯光、插座、窗帘、空调、电视等设备控制
- **状态反馈**：实时反馈设备开关状态、控制模式、控制时间
- **本地控制**：断网时仍可本地控制（应急控制）
- **远程控制**：通过云端远程控制设备
- **门禁管理**：RFID门禁读卡、电控锁控制
- **传感器数据**：采集客房环境数据上报云端

#### 硬件组成
- **主控芯片**：ESP32-WROOM-32（双核240MHz，WiFi+蓝牙）
- **执行器**：
  - 继电器模块（4路，10A/250VAC，灯光/插座/窗帘/备用）
  - 红外发射模块（空调/电视红外控制）
  - 红外接收模块（红外学习）
  - 电控锁（门禁控制）
- **传感器**：
  - RFID模块（RC522，13.56MHz，门禁读卡）
- **状态指示**：RGB LED（WS2812B×5颗）
- **电源**：DC 5V/1A，Micro USB

#### 通信方式
- **MQTT**：与云端服务器通信
- **HTTP**：数据上报

#### 成本预算
- **经济型**：¥90/台
- **标准型**：¥145/台（含更多传感器和功能）

## 📡 系统通信架构

### 通信层级
1. **第一层**：前端 ↔ 后端（HTTP/WebSocket）
2. **第二层**：前台管理端 ↔ 前台电脑（USB串口）
3. **第三层**：楼控/客房端 ↔ 后端（MQTT/HTTP）

### 通信协议
- **前端-后端**：HTTP（RESTful API）+ WebSocket（实时通信）
- **前台管理端-前台电脑**：USB串口（自定义JSON协议）
- **楼控/客房端-后端**：MQTT（设备通信）+ HTTP（数据上报）
- **配网方式**：蓝牙BLE（初始配网）+ USB串口（调试配网）

### 数据流向
```
客房端 → 后端 → 前端（手机/Web）
楼控 → 后端 → 前端（手机/Web）
前台管理端 → 前台电脑 → 后端 → 前端（手机/Web）
前端（手机/Web）→ 后端 → 楼控/客房端
前端（手机/Web）→ 后端 → 前台管理端（通过前台电脑）
```

### 用户交互原则
- **前台管理端不直接参与用户交互**：前台管理端没有显示屏，通过USB与前台电脑协同工作
- **后端作为通信枢纽**：所有设备（前台管理端、楼控、客房端、前端）都与后端直接通信
- **前台管理端通过LED指示状态**：前台管理端只通过RGB灯带和蜂鸣器提供物理反馈
- **前后端分离**：前端负责用户界面，后端负责业务逻辑和数据处理

## 💰 成本控制

### 硬件成本预算（按50间客房计算）

| 配置方案 | 前台管理端 | 楼控（3台） | 客房终端（50间） | **总计** |
|---------|------------|-------------|----------------|----------|
| **经济型** | ¥84 | ¥105 × 3 = ¥315 | ¥90 × 50 = ¥4500 | **¥4900** |
| **标准型** | ¥84 | ¥105 × 3 = ¥315 | ¥145 × 30 + ¥90 × 20 = ¥6750 | **¥7150** |
| **豪华型** | ¥84 | ¥105 × 3 = ¥315 | ¥145 × 50 = ¥7250 | **¥7650** |

### 成本优化策略
- 使用ESP32作为主控芯片，性价比高
- 选用成熟可靠的传感器模块
- 模块化设计，硬件可复用
- 批量采购降低成本
- 精简非核心功能
- 一机多用，避免硬件浪费
- 前台管理端采用USB外设模式，移除冗余硬件（OLED/麦克风/扬声器/WiFi），节省48%成本

## 🔧 技术栈

### 硬件技术栈
- **主控芯片**：
  - 前台管理端：ESP32-S2-MINI / STM32F103C8T6（USB从机模式）
  - 楼控/客房端：ESP32-WROOM-32（240MHz，双核，WiFi+蓝牙）
- **传感器**：DHT11、MQ-135、光敏电阻、HC-SR501、门磁传感器、RC522 RFID
- **执行器**：继电器、RGB LED（WS2812B）、电控锁、红外发射
- **通信模块**：
  - 前台管理端：USB串口（无WiFi）
  - 楼控/客房端：ESP32内置WiFi、蓝牙BLE
- **外部接口**：RS485（消防）、UART（公安）

### 软件技术栈
- **嵌入式开发**：Arduino IDE、ESP-IDF
- **后端开发**：Node.js + Express
- **数据库**：MySQL + Redis
- **前端开发**：Vue.js 3 + Ant Design Vue
- **移动端开发**：Vue.js PWA（手机端H5/小程序）
- **通信协议**：MQTT、HTTP、WebSocket、USB串口协议
- **AI服务**：智谱GLM-4-Flash（统一后端服务）
