### 设备角色定义

#### 主控中心
- **功能定位**：系统核心，负责本地交互、语音助理、系统管理
- **核心职责**：
  - 与后端服务器通信，上传数据、接收指令
  - 本地人机交互（按键、显示屏、扬声器）
  - AI语音助理功能（语音识别、语音反馈）
  - 系统状态管理和监控
  - 本地设备控制（可选功能）

#### 分控1：环境监测与设备控制
- **功能定位**：环境数据采集和家居设备控制
- **核心职责**：
  - 温湿度、空气质量、光照强度监测
  - 智能灯光控制（RGB LED调光调色）
  - 智能插座控制（继电器开关电器）
  - 红外遥控学习（控制空调、电视等）
  - 与后端服务器通信，上传数据、接收指令

#### 分控2：安防管理终端
- **功能定位**：门禁管理和安全监测
- **核心职责**：
  - RFID门禁系统（刷卡开门）
  - 远程开门（手机端控制）
  - 门窗状态监测
  - 人体检测入侵报警
  - 电控锁控制
  - 与后端服务器通信，上传数据、接收指令

## 📡 通信架构

### 通信层级
1. **第一层**：前端 ↔ 后端（HTTP/WebSocket）
2. **第二层**：主控 ↔ 后端（MQTT/HTTP）
3. **第三层**：分控1 ↔ 后端（MQTT/HTTP）
4. **第四层**：分控2 ↔ 后端（MQTT/HTTP）

### 通信协议
- **前端-后端**：HTTP（RESTful API）+ WebSocket（实时通信）
- **主控-后端**：MQTT（设备通信）+ HTTP（数据上报）
- **分控-后端**：MQTT（设备通信）+ HTTP（数据上报）
- **分控配网**：蓝牙BLE（初始配网）+ USB串口（调试配网）
