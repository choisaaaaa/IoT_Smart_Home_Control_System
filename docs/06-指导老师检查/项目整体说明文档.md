## 📋 项目概览

### 项目名称
**"智联酒店"智慧酒店物联网控制系统**

### 项目类型
物联网应用 - 行业应用小类（酒店行业）

### 项目目标
开发一个低成本、高完成度的智慧酒店物联网控制系统，采用模块化、可复用的硬件设计，支持多房间批量组网和云端联控，实现客房智能化、运营管理数字化、能源管理精细化的智慧酒店整体解决方案。

### 行业背景与趋势

#### 行业背景
智慧酒店是以数字化与网络化技术为核心，通过物联网、人工智能等技术构建智能化管理体系的新型酒店模式，旨在提升住客服务体验与运营管理效率。2025年，随着AI与具身智能技术的深入应用，酒店行业正从功能智能化向场景智能体阶段演进，通过"感知—认知—决策—执行—反馈"的AI调度闭环构建智能服务模式。2026年，智慧酒店领域在人才培养与产教融合方面呈现新趋势，酒店业市场在文旅热潮、消费升级及大型活动等多重因素驱动下持续发展。

#### 行业痛点
1. **入住流程繁琐**：传统酒店入住需要排队登记、领房卡，效率低
2. **能源浪费严重**：客房无人时空调、灯光持续运行，能耗高
3. **安全管理薄弱**：传统门禁系统安全性不足，无法远程监控
4. **运营成本高**：人工巡检成本高，设备故障发现不及时
5. **客户体验差**：缺乏个性化服务，无法满足多样化需求

#### 解决方案
1. **无接触入住**：手机APP定位+授权，自动开房门，提升效率
2. **智能节能**：无人自动断电、空调节能控制，降低能耗30%+
3. **智能安防**：RFID门禁、远程开门、门窗监测，提升安全性
4. **远程监控**：云端集中监控、预测性维护，降低运营成本
5. **个性化服务**：语音控制、场景模式、客户偏好记录，提升体验

### 项目定位
本项目面向中小型酒店及民宿领域，提供低成本、易部署的智慧酒店解决方案，支持无接触操作与个性化场景设置（如自动调节室温、灯光），实现"感知—认知—决策—执行—反馈"的AI调度闭环。

### 技术栈

#### 前端技术栈
- **Web前端**：Vue.js 3 + Ant Design Vue + ECharts
- **移动端**：Flutter App
- **状态管理**：Pinia
- **HTTP客户端**：Axios
- **WebSocket客户端**：Socket.io-client

#### 后端技术栈
- **后端框架**：Node.js + Express
- **数据库**：MySQL（主数据库）+ Redis（缓存）
- **消息队列**：MQTT Broker（Mosquitto）
- **实时通信**：WebSocket（Socket.io）
- **AI服务**：语音识别API（阿里云）、语音合成API（讯飞/阿里）、大语言模型（智谱 GLM-4-Flash）

#### 嵌入式技术栈
- **主控芯片**：ESP32-WROOM-32（240MHz，双核，WiFi+蓝牙）
- **开发环境**：Arduino IDE / ESP-IDF
- **通信协议**：MQTT、HTTP、WebSocket

## 🚀 部署方案

### 方案1：本地开发 + Docker部署（推荐）

**适合场景**：开发阶段，环境隔离，易于调试

#### 部署步骤

```bash
# 1. 克隆项目
git clone https://github.com/your-username/iot-smart-hotel.git
cd iot-smart-hotel

# 2. 配置环境变量
cd backend
cp .env.example .env
# 编辑 .env 文件，填写 API Keys

# 3. 启动所有服务
cd ..
docker-compose up -d

# 4. 查看服务状态
docker-compose ps

# 5. 访问后端 API
curl http://localhost:3000/api/v1/health
```

#### 服务列表

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端服务 | 3000 | Node.js后端API服务 |
| 前端服务 | 80 | Vue3前端服务 |
| MySQL | 3306 | MySQL数据库服务 |
| Redis | 6379 | Redis缓存服务 |
| MQTT Broker | 1883 | MQTT消息 broker服务 |
| WebSocket | 3001 | WebSocket实时通信服务 |

### 方案2：云服务器部署（比赛演示）

**适合场景**：比赛演示，远程访问，环境一致

#### 部署步骤

```bash
# 1. 在云服务器上执行
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# 2. 克隆项目
git clone https://github.com/your-username/iot-smart-hotel.git
cd iot-smart-hotel

# 3. 配置环境变量
cd backend
cp .env.example .env
# 编辑 .env 文件，填写 API Keys

# 4. 启动服务
cd ..
docker-compose up -d

# 5. 配置 Nginx 反向代理（可选）
# 参考部署指南第 2.6 节

# 6. 获取服务器公网 IP
curl ifconfig.me
```

#### 云服务器配置

| 配置项 | 推荐配置 | 说明 |
|--------|----------|------|
| CPU | 2核 | 满足系统运行需求 |
| 内存 | 4GB | 满足数据库和缓存需求 |
| 存储 | 50GB | 满足数据存储需求 |
| 带宽 | 5Mbps | 满足远程访问需求 |

### 方案3：跨平台开发环境（本地Windows + 云Ubuntu）

**适合场景**：团队协作开发，环境一致性要求高

#### 开发流程

```
Windows本地开发 (Docker)  →  Git提交  →  Ubuntu云服务器部署 (Docker)
   (VS Code + Docker)        (代码管理)       (Docker Compose)
```

#### 本地开发（Windows）

```bash
# 1. 安装Docker Desktop
# 下载: https://www.docker.com/products/docker-desktop/

# 2. 克隆项目
git clone https://github.com/your-username/iot-smart-hotel.git
cd iot-smart-hotel

# 3. 启动开发环境
docker-compose up -d

# 4. 本地开发
cd backend
npm install
npm run dev
```

#### 云服务器部署（Ubuntu）

```bash
# 1. 云服务器初始化
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 2. 克隆项目
git clone https://github.com/your-username/iot-smart-hotel.git
cd iot-smart-hotel

# 3. 部署
docker-compose up -d
```

## 📊 性能指标

### 系统性能要求
- **响应时间**：设备控制响应时间 < 1秒
- **数据传输**：传感器数据上传频率 1-10秒
- **并发连接**：支持10个客户端同时连接
- **系统稳定性**：连续运行7天无故障
- **数据准确性**：传感器误差 < 5%

### 硬件性能要求
- **工作温度**：0-50℃
- **工作湿度**：20-90%RH
- **供电电压**：5V DC
- **功耗**：正常工作 < 5W
- **通信距离**：WiFi < 50米，蓝牙 < 10米

## 💡 创新点

### 1. 模块化硬件设计
- 一机多用：同一终端支持客房控制、环境监测、安防监控三种模式
- 模块复用：硬件组件可在不同模式间复用，降低硬件成本
- 灵活扩展：支持按需配置，适应不同规模酒店需求

### 2. AI语音助理
- 集成智谱GLM-4-Flash大模型，提供智能对话能力
- 支持语音识别（阿里云ASR）和语音合成（讯飞TTS）
- 实现自然语言交互，提升用户体验

### 3. 云端联控
- 支持批量组网，单个系统可管理100+房间
- 实现远程集中控制，降低人工巡检成本
- 提供预测性维护提醒，提高设备利用率

### 4. 节能优化
- 无人房间自动断电，降低能耗30%+
- 空调智能节能控制
- 灯光自适应调节
- 实时能耗统计与分析
