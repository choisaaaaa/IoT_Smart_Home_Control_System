# 物联网智能家居控制系统 - 整体需求

## 📋 项目概述

### 项目名称
**"智联万物"分布式智能家居控制系统**

### 项目类型
物联网应用 - 数字生活小类

### 项目目标
开发一个低成本、高完成度的分布式智能家居控制系统，采用主控+分控的分布式架构，实现环境监测、安防管理、设备控制、远程管理等功能，为大学生计算机设计大赛准备技术材料。

### 核心特点
- **分布式架构**：主控+分控设计，模块化、易扩展
- **低成本**：硬件总成本控制在¥200-¥400
- **高完成度**：20天内完成完整可演示系统
- **系统完整**：包含硬件、软件、云端、移动端全栈
- **易于实现**：使用成熟技术栈，学习曲线平缓
- **灵活配置**：网络参数可在线修改，无需重新烧录

## 🎯 设备角色定义

### 主控中心
- **功能定位**：系统核心，负责本地交互、语音助理、系统管理
- **核心职责**：
  - 与后端服务器通信，上传数据、接收指令
  - 本地人机交互（按键、显示屏、扬声器）
  - AI语音助理功能（语音识别、语音反馈）
  - 系统状态管理和监控
  - 本地设备控制（可选功能）
- **硬件组成**：ESP32主控 + 扬声器 + 显示屏 + 按键 + 蜂鸣器
- **通信方式**：MQTT（与后端通信）+ HTTP（数据上报）

### 分控1：环境监测与设备控制
- **功能定位**：环境数据采集和家居设备控制
- **核心职责**：
  - 温湿度、空气质量、光照强度监测
  - 智能灯光控制（RGB LED调光调色）
  - 智能插座控制（继电器开关电器）
  - 红外遥控学习（控制空调、电视等）
  - 蓝牙配网功能
  - USB串口调试功能
  - 与后端服务器通信，上传数据、接收指令
- **硬件组成**：ESP32 + DHT11 + MQ-135 + 光敏电阻 + 继电器 + RGB LED + 红外发射+红外接收
- **通信方式**：MQTT（与后端通信）+ HTTP（数据上报）

### 分控2：安防管理终端
- **功能定位**：门禁管理和安全监测
- **核心职责**：
  - RFID门禁系统（刷卡开门）
  - 远程开门（手机端控制）
  - 门窗状态监测
  - 人体检测入侵报警
  - 摄像头监控（可选）
  - 电控锁控制
  - 蓝牙配网功能
  - USB串口调试功能
  - 与后端服务器通信，上传数据、接收指令
- **硬件组成**：ESP32 + RFID模块 + 门磁传感器 + 人体传感器 + 蜂鸣器 + 电控锁 + 摄像头（可选）
- **通信方式**：MQTT（与后端通信）+ HTTP（数据上报）
  - USB串口调试功能
- **硬件组成**：ESP32 + RFID模块 + 门磁传感器 + 人体传感器 + 蜂鸣器 + 电控锁 + 摄像头（可选）

## 📡 系统通信架构

### 通信层级
1. **第一层**：前端 ↔ 后端（HTTP/WebSocket）
2. **第二层**：主控 ↔ 后端（MQTT/HTTP）
3. **第三层**：分控1 ↔ 后端（MQTT/HTTP）
4. **第四层**：分控2 ↔ 后端（MQTT/HTTP）

### 通信协议
- **前端-后端**：HTTP（RESTful API）+ WebSocket（实时通信）
- **主控-后端**：MQTT（设备通信）+ HTTP（数据上报）
- **分控-后端**：MQTT（设备通信）+ HTTP（数据上报）
- **分控配网**：蓝牙BLE（初始配网）+ USB串口（调试配网）

### 数据流向
```
分控1 → 后端 → 前端（手机/Web）
分控2 → 后端 → 前端（手机/Web）
主控 → 后端 → 前端（手机/Web）
前端（手机/Web）→ 后端 → 主控
前端（手机/Web）→ 后端 → 分控1
前端（手机/Web）→ 后端 → 分控2
```

### 用户交互原则
- **分控不直接参与用户交互**：分控设备没有物理按键、显示屏等交互界面
- **后端作为通信枢纽**：所有设备（主控、分控、前端）都与后端直接通信
- **分控通过LED指示状态**：分控只通过LED颜色指示工作状态，不提供其他交互
- **前后端分离**：前端负责用户界面，后端负责业务逻辑和数据处理

## 🔧 技术栈

### 硬件技术栈
- **主控芯片**：ESP32-WROOM-32（240MHz，双核，WiFi+蓝牙）
- **传感器**：DHT11、MQ-135、光敏电阻、HC-SR501、门磁传感器
- **执行器**：继电器、RGB LED、电控锁、红外发射
- **通信模块**：ESP32内置WiFi、蓝牙BLE

### 软件技术栈
- **嵌入式开发**：Arduino IDE、ESP-IDF
- **后端开发**：Node.js + Express
- **数据库**：SQLite / MySQL
- **前端开发**：Vue.js 3 + Element Plus
- **移动端开发**：Flutter / 微信小程序
- **通信协议**：MQTT、HTTP、WebSocket

## 📊 性能指标

### 系统性能要求
- **响应时间**：设备控制响应时间 < 1秒
- **数据传输**：传感器数据上传频率 1-10秒
- **并发连接**：支持10个客户端同时连接
- **系统稳定性**：连续运行7天无故障
- **数据准确性**：传感器误差 < 5%

### 硬件性能要求
- **工作温度**：0-50℃
- **工作湿度**：20-90%RH
- **供电电压**：5V DC
- **功耗**：正常工作 < 5W
- **通信距离**：WiFi < 50米，蓝牙 < 10米
