### 硬件功能说明

#### 客房端功能定位
- **设备控制**：控制客房设备（灯光、插座、窗帘等）
- **状态反馈**：实时反馈设备状态
- **本地控制**：断网时仍可本地控制
- **远程控制**：通过云端远程控制
- **状态指示**：RGB LED显示工作状态
- **实物演示**：集成门锁、灯光、窗帘、空调、电视等演示功能
- **🚔客房安全**：门禁通行日志记录、**住客SOS一键求助**
- **🚒消防响应（GB 25506合规）**：接收疏散指令→自动开锁+亮灯+关空调+蜂鸣报警

#### 2. 核心硬件模块与演示功能

##### 2.1 主控板（ESP32-WROOM-32）
- **功能**：系统核心控制，数据处理，通信管理
- **特性**：
  - 双核Xtensa LX6，240MHz
  - 4MB Flash，520KB RAM
  - WiFi 802.11 b/g/n、蓝牙4.2 BLE
  - GPIO、UART、I2C、SPI、ADC、DAC接口

##### 2.2 继电器模块（4路，10A/250VAC）
- **功能**：客房设备控制
- **继电器分配**：
  - 继电器1：灯光控制 → 驱动RGB灯带
  - 继电器2：插座控制 → 备用或驱动小负载
  - 继电器3：窗帘电机控制 → 驱动舵机电源
  - 继电器4：门锁控制 → 驱动电磁锁

##### 2.3 OLED显示屏（0.96寸I2C）
- **功能**：显示设备状态和演示信息
- **显示内容**：
  - 当前设备开关状态
  - 门锁状态（已锁/已开）
  - 灯光亮度/颜色模式
  - 窗帘开合百分比
  - 空调温度/模式
  - **演示时显示**：当前操作指令和反馈结果

##### 2.4 按键模块（4键独立按键 + SOS求助键）
- **功能**：本地控制和演示切换
- **按键功能**：
  - 按键1：灯光开关
  - 按键2：窗帘开关
  - 按键3：门锁测试（模拟刷卡）
  - 按键4：**场景模式切换 / 演示流程启动**
  - **SOS求助键**：**🚔住客一键求助→上报楼控/前台**

##### 2.5 蜂鸣器（有源蜂鸣器，5V）
- **功能**：声音反馈 + 🚒火警警报
- **演示用途**：
  - 开门成功提示音
  - 设备操作确认音
  - 异常报警声
- **🚒火警音效**：
  | 场景 | 音效 | 说明 |
  |------|------|------|
  | 收到疏散指令 | 连续急促蜂鸣(消防警报) | 火灾时提醒住客撤离 |
  | SOS求助确认 | "滴—滴—"两声长 | SOS信号已发送 |

##### 2.6 RGB LED(状态)（WS2812B×5颗）
- **功能**：客房工作状态指示
- **状态指示**：
  - 绿色常亮：正常工作
  - 蓝色闪烁：WiFi连接中
  - 红色闪烁：通信异常
  - 彩色渐变：演示进行中
  - **🚒红色快闪(2Hz)**：**收到火警疏散指令，紧急模式**

##### 2.7 RFID模块(RC522) + 电磁锁 → 🔐 门锁演示
- **功能**：RFID门禁控制
- **演示场景**：
  - 刷有效房卡 → 蜂鸣器"滴"一声 → 电磁锁弹开 → OLED显示"✓ 开门成功" → 绿灯亮
  - 倒计时5秒后自动上锁 → 电磁锁吸合 → OLED显示"🔒 门已上锁"
  - 刷无效卡/未登记卡 → 蜂鸣器连续报警 → OLED显示"✗ 非法卡片" → 红灯闪烁
  - 接收远程开门指令（来自前台管理端/APP）→ 与刷卡效果相同
- **MQTT上报**：每次刷卡/开门/上锁事件实时上报云端

##### 2.8 RGB灯带(WS2812B×8颗) → 💡 灯光演示
- **功能**：模拟客房智能灯光系统
- **演示场景**：
  - 按键1/远程指令 → 灯带开关（渐亮/渐灭效果）
  - PWM亮度调节 → 0%-100%平滑调节
  - 颜色切换 → 红/绿/蓝/暖白/RGB全彩循环
  - 场景模式（按住按键4切换）：
    - 🛏️ 睡眠模式：暖黄色，10%亮度，30秒渐暗至关闭
    - 📖 阅读模式：冷白色，80%亮度
    - 🎬 娱乐模式：RGB彩色呼吸灯效果
    - 🚪 离房模式：全部关闭
- **OLED同步显示**：当前亮度值、颜色名称、场景模式名

##### 2.9 舵机(SG90) → 🪟 窗帘演示
- **功能**：模拟电动窗帘开合
- **演示场景**：
  - 按键2/远程指令"打开窗帘" → 舵机从90°转到0°（约1秒）→ OLED显示"窗帘: 100%打开"
  - "关闭窗帘" → 舵机从0°转回90° → OLED显示"窗帘: 已关闭"
  - "窗帘开50%" → 舵机转到45° → OLED显示"窗帘: 50%"
  - 到达限位后自动停止（软件保护）
- **物理表现**：舵机摇臂摆动直观展示窗帘开合动作

##### 2.10 红外发射模块 → ❄️ 空调演示 / 📺 电视演示
- **功能**：红外遥控信号发送
- **空调演示场景**：
  - 发送开机红外码 → OLED显示"空调: ▶ 制冷 26℃"
  - 温度+/- → 调节16-30℃，OLED实时更新
  - 模式切换 → 制冷/制热/送风/除湿，OLED显示对应图标
  - 风速调节 → 低/中/高/自动
  - （如有真实空调在演示现场则实际控制，否则OLED模拟显示）
- **电视演示场景**：
  - 发送开机码 → OLED显示"电视: ▶ 已开启"
  - 音量+/- / 频道+/- → OLED显示当前值
  - 支持多品牌红外码库内置（格力/美的/海尔/小米/海信/TCL等）
- **红外学习功能**：通过红外接收模块(VS1838B)可学习新的遥控器红外码并存储到Flash

### 6. 🚔 客房安全功能（公安合规）

#### 6.1 门禁通行日志记录
- **法规依据**：《旅馆业治安管理办法》要求门禁通行记录可追溯
- **记录内容**：卡号、时间、结果（通过/拒绝）、房间号
- **存储方式**：本地Flash暂存 + MQTT实时上报楼控/前台
- **保留期**：≥90天（由楼控/前台统一存储，客房端负责采集上报）
- **上报Topic**：`hotel/room/{room_id}/door_event`

#### 6.2 住客SOS一键求助
- **触发方式**：SOS求助键短按或长按
- **联动动作**：
  1. 蜂鸣器"滴—滴—"两声长（确认信号已发送）
  2. OLED显示"🆘 SOS已发送! 请等待..."
  3. RGB LED橙色闪烁
  4. MQTT消息 `hotel/room/{room_id}/sos` → 上报楼控 → 转发前台管理端
  5. （如前台在线）→ 前台OLED显示"XXX房SOS求助" + 扬声器语音提示
- **复位条件**：再次按SOS 或 前台远程解除
- **演示场景**：按下SOS键 → 观察蜂鸣器+LED+OLED变化 → 串口显示MQTT消息内容

#### 6.3 非法入侵检测
- **触发条件**：
  - 连续3次无效刷卡（未登记卡/过期卡/黑名单卡）
  - 强行开门检测（电磁锁被强制断电/物理破坏模拟）
- **联动动作**：
  - 蜂鸣器连续急促报警
  - OLED显示"⚠️ 非法入侵!"
  - RGB LED红闪
  - MQTT `hotel/room/{room_id}/intrusion` 上报楼控和前台

### 7. 🚒 消防响应功能（符合GB 25506《消防控制室通用技术要求》）

#### 7.1 接收疏散指令
- **接收来源**：MQTT Topic `hotel/emergency/evacuate`
- **触发条件**：收到前台管理端或楼控广播的疏散指令
- **Payload格式**：
  ```json
  {
    "type": "fire_evacuate",
    "source": "front_desk",
    "floor": "all",
    "timestamp": 1743768000,
    "message": "立即疏散",
    "actions": ["door_unlock", "light_on", "curtain_open", "ac_off"]
  }
  ```

#### 7.2 自动执行消防响应序列
```
收到疏散指令(T+0s)
    │
    ├─→ [T+0.1s] 声光报警
    │     ├─ 蜂鸣器：连续消防警报声
    │     ├─ RGB LED(状态)：红色快闪(2Hz)
    │     └─ OLED："🚒 火灾警报! 请立即撤离!"
    │
    ├─→ [T+0.3s] 门锁解锁
    │     └─ 继电器4释放 → 电磁锁弹开（便于逃生）
    │
    ├─→ [T+0.5s] 灯光全亮
    │     └─ RGB灯带全白100%亮度（照亮逃生路径）
    │
    ├─→ [T+0.7s] 窗帘全开
    │     └─ 舵机转到90°（增加采光/排烟通道）
    │
    ├─→ [T+1.0s] 空调关闭
    │     └─ 红外发射空调关机码（防止火势蔓延）
    │
    └─→ [T+1.5s] 上报确认
          └─ MQTT: hotel/room/{room_id}/evacuate_ack
              → 确认已执行全部疏散动作
```

#### 7.3 复位条件
- 收到复位指令（`hotel/emergency/reset`）
- 消防主机复位信号（经楼控转发）
- 本地按键4长按5秒（紧急手动复位）

#### 7.4 备用电源接口预留
- **目的**：火灾时市电可能中断，客房端需继续运行至少30分钟
- **接口设计**：电源模块预留DC输入端子，可接入备用电池
- **切换逻辑**：市电正常时充电；市电中断时无缝切换

## 🔧 硬件调试功能

### 1. USB串口调试
- **功能描述**：通过USB串口进行调试
- **功能要求**：
  - 串口波特率：115200
  - 支持日志输出
  - 支持命令调试
  - 支持固件升级

### 2. WiFi配网
- **功能描述**：通过WiFi进行配网
- **功能要求**：
  - 支持热点模式配网
  - 支持STA模式配网
  - 支持配网状态显示
  - 支持配网日志记录
