# 智慧酒店物联网控制系统 - 三层硬件架构设计说明

## 📋 架构概述

本项目采用"前台管理端-楼控-客房端"三层硬件架构，实现智慧酒店的集中管理、环境监测和客房控制功能。

### 架构优势
- **分层清晰**：前台管理端负责协调，楼控负责监测，客房端负责控制
- **功能专一**：每个层级功能明确，易于开发和维护
- **轻硬重软**：前台管理端作为电脑外设，软件功能由前台电脑承担，大幅降低硬件复杂度
- **成本优化**：前台管理端成本降低48%（¥150→¥84），适合大规模部署
- **灵活部署**：楼控可选部署，根据酒店需求配置
- **云端协同**：所有数据上报云端，支持远程监控和管理

### 🆕 前台管理端新定位：USB外设模式

> **核心思路**：前台管理端硬件直接连接前台电脑，通过USB接口通信。
> **显示、语音识别、网络通信、数据存储等"重软件"功能全部由前台电脑承担**，
> 本设备仅保留物理I/O组件（RFID读卡、状态灯墙、SOS按键、蜂鸣器、RS485/UART接口）。

## 🏗️ 三层架构详细设计

### 1️⃣ 前台管理端（酒店级协调 + USB外设模式）

#### 🆕 功能定位（USB外设版）
- **硬件层（本设备）**：物理I/O组件，负责房卡读写、状态可视化、SOS报警、消防/公安接口
- **软件层（前台电脑）**：GUI界面、AI语音助理、MQTT通信、数据存储、业务逻辑
- **通信方式**：USB串口通信，自定义协议帧格式
- **供电方式**：USB取电（5V），无需独立电源

#### 核心功能

##### 1.1 酒店级协调（软件层 - 前台电脑）
- **功能描述**：统一管理所有客房终端
- **功能要求**：
  - 支持100+客房终端管理
  - 批量组网功能
  - 分区控制功能
  - 设备状态汇总
- **说明**：作为酒店级控制核心，由前台电脑软件实现

##### 1.2 本地交互（软硬件协同）
- **功能描述**：本地酒店状态显示和操作
- **职责分工**：
  - **🖥️ 前台电脑**：GUI显示酒店总客房数、已入住客房数、能源消耗
  - **🔌 本设备**：按键触发操作、蜂鸣器提示音、RGB LED状态指示
- **说明**：显示由电脑承担，物理交互由本设备承担

##### 1.3 🤖 智能酒店管家系统（软件层 - 前台电脑 + 手机端APP）

> **定位升级**：从单纯的"语音控制设备"升级为**"智能酒店管家"**，
> 提供完整的酒店服务体验，支持**前台电脑端（管理员）**和**手机端（住客/员工）**双平台访问。

- **功能描述**：AI语音交互 + 酒店服务管理
- **系统架构**：
  ```
  ┌──────────────────────┐    ┌──────────────────────────┐
  │   🖥️ 前台电脑端      │    │   📱 手机端(住客/员工)    │
  │   (管理员)            │◄──►│                          │
  │ ·入住/退房管理       │MQTT│ ·客房服务请求            │
  │ ·房卡/工单管理       │API │ ·联系前台/送物/报修       │
  │ ·数据分析报表        │    │ ·设备控制(灯光/空调等)   │
  └──────────┬───────────┘    └────────────┬─────────────┘
             └──────────┬──────────────────┘
                        ▼
              ┌─────────────────┐
              │  ☁️ 云端服务器     │
              │  AI语音/NLP引擎   │
              │  业务逻辑/消息推送│
              └─────────────────┘
  ```

- **核心功能模块**：

| 功能域 | 子功能 | 前台电脑端 | 手机端(住客) | 说明 |
|--------|--------|:----------:|:------------:|------|

- **职责分工**：
  - **🖥️ 前台电脑**：管理员工作台、工单处理中心、数据分析、语音交互（麦克风+ASR+TTS）
  - **📱 手机端**：住客服务入口、设备控制、服务请求、消息接收
  - **☁️ 云端服务器**：AI引擎、业务逻辑、消息推送、数据同步
  - **🔌 本设备**：无语音相关硬件（已移除），仅提供物理I/O反馈
- **说明**：软件系统完全由前台电脑+云端+手机端协同实现

##### 1.4 数据汇总与存储（软件层 - 前台电脑）
- **功能描述**：收集所有客房数据并存储
- **职责分工**：
  - **🖥️ 前台电脑**：数据收集、数据库存储（日志≥90天）、数据分析
  - **🔌 本设备**：无存储模块（已移除Flash）
- **说明**：数据存储完全由前台电脑承担

##### 1.5 指令下发（软件层 - 前台电脑）
- **功能描述**：接收云端/本地指令并转发
- **职责分工**：
  - **🖥️ 前台电脑**：MQTT客户端通信、指令转发、批量控制
  - **🔌 本设备**：无WiFi模块（已移除）
- **说明**：网络通信完全由前台电脑承担

##### 1.6 房卡管理（硬件层 - 本设备）
- **功能描述**：房卡读卡、写卡、注销
- **硬件实现**：
  - RFID模块(RC522)读取/写入Mifare卡
  - 支持房卡制作（写入房号、有效期、权限等级）
  - 支持房卡注销（退房时清空数据）
  - 支持黑名单管理
- **说明**：**这是本设备的核心功能之一，必须保留物理硬件**

##### 1.7 客房状态可视化墙（硬件层 - 本设备）
- **功能描述**：RGB灯带实时显示各客房状态
- **硬件实现**：
  - WS2812B×8颗LED，每颗代表一间房
  - 颜色编码：白色=空置、绿色=已入住、红色=SOS报警、黄色=清洁中、蓝色=维修中
  - 支持呼吸动画（入住时）、闪烁动画（报警时）
- **说明**：**物理展示墙，直观展示酒店运营状态**

##### 1.8 SOS一键报警（硬件层 - 本设备）
- **功能描述**：110一键报警功能
- **硬件实现**：
  - 独立SOS报警键（物理按键）
  - 蜂鸣器播放警报音效
  - RGB LED闪烁报警
  - 通过USB通知前台电脑软件上报MQTT
- **说明**：**符合公安要求，必须有物理按键**

##### 1.9 消防主机接口（硬件层 - 本设备）
- **功能描述**：对接消防主机（RS485/MODBUS协议）
- **硬件实现**：
  - RS485模块(MAX3485)
  - MODBUS协议通信
  - 接收火警信号、设备状态轮询
  - 通过USB转发给前台电脑软件
- **说明**：**符合国标GB 50116消防联动要求**

##### 1.10 公安终端接口（硬件层 - 本设备）
- **功能描述**：预留身份证读卡器/公安终端接口
- **硬件实现**：
  - UART扩展口(4Pin接插端子)
  - 可对接身份证读卡器
  - 可对接公安终端
  - 通过USB转发给前台电脑软件
- **说明**：**符合《旅馆业治安管理办法》实名登记要求**

### 2️⃣ 楼控（环境监测与服务支持）

#### 功能定位
- 环境监测：温湿度、空气质量、光照强度
- 数据采集：实时采集环境数据
- 数据上报：将环境数据上报云端
- 状态指示：RGB LED显示工作状态
- 辅助控制：支持灯光、插座等基础设备控制

#### 核心功能

##### 2.1 环境监测
- **功能描述**：实时监测环境参数
- **功能要求**：
  - 温湿度监测（DHT11）
  - 空气质量监测（MQ-135）
  - 光照强度监测（光敏电阻）
  - 数据实时上报
- **说明**：环境监测中心

##### 2.2 数据采集
- **功能描述**：实时采集环境数据
- **功能要求**：
  - 采集频率：1-10秒
  - 数据准确性：误差 < 5%
  - 数据完整性：无丢失
- **说明**：数据采集中心

##### 2.3 数据上报
- **功能描述**：将环境数据上报云端
- **功能要求**：
  - 上报协议：MQTT
  - 上报频率：1-10秒
  - 数据格式：JSON
  - 断线重连：自动重连
- **说明**：数据上报中心

##### 2.4 状态指示
- **功能描述**：RGB LED显示工作状态
- **功能要求**：
  - 正常工作：绿色LED
  - 通信异常：红色LED
  - 数据采集中：蓝色LED
- **说明**：本地状态指示

##### 2.5 辅助控制
- **功能描述**：支持灯光、插座等基础设备控制
- **功能要求**：
  - 2路继电器控制
  - 本地控制（断网可用）
  - 远程控制
  - 定时控制
- **说明**：辅助设备控制

### 3️⃣ 客房端（客房设备控制与状态反馈）

#### 功能定位
- 设备控制：灯光、插座等设备控制
- 状态反馈：实时反馈设备状态
- 本地控制：断网时仍可本地控制
- 远程控制：通过云端远程控制
- 状态指示：RGB LED显示工作状态

#### 核心功能

##### 3.1 设备控制
- **功能描述**：控制客房设备（灯光、插座等）

## 🎯 部署建议

### 小型酒店（10-30间房）
- **前台管理端**：1个（主控中心）
- **楼控**：1-2个（大堂+走廊）
- **客房端**：每间房1个（10-30个）

### 中型酒店（30-100间房）
- **前台管理端**：1-2个（主控+备用）
- **楼控**：3-5个（大堂+各楼层）
- **客房端**：每间房1个（30-100个）

### 大型酒店（100+间房）
- **前台管理端**：2-3个（主控+备用+分区控制）
- **楼控**：5-10个（大堂+各楼层+公共区域）
- **客房端**：每间房1个（100+个）
