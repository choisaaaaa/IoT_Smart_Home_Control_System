# 智慧酒店物联网控制系统 - 三层架构客房端需求

## 📋 文档概述

### 文档说明
本文档描述智慧酒店物联网控制系统的三层架构中客房端硬件需求，包括硬件组成、功能需求、接口定义、调试功能等。

### 设备定位
客房端是智慧酒店三层架构的客房设备控制与状态反馈设备，采用"轻硬件+重软件"设计理念，硬件仅保留核心功能（继电器控制、红外遥控、RFID门禁），更多功能通过软件实现。

### 设计理念
- **硬件精简**：仅保留核心功能硬件，降低成本、缩短开发周期
- **软件增强**：通过酒店APP实现丰富功能，提升用户体验
- **云端AI**：环境监测、节能优化、AI管家等AI算法在云端实现
- **大规模部署**：客房终端成本低（¥90），适合多房间部署

## 🔧 硬件组成（精简版）

### 1. 核心控制模块

#### 1.1 ESP32主控板
- **型号**：ESP32-WROOM-32
- **参数**：
  - CPU：双核Xtensa LX6，240MHz
  - Flash：4MB
  - RAM：520KB
  - 无线：WiFi 802.11 b/g/n、蓝牙4.2 BLE
  - 接口：GPIO、UART、I2C、SPI、ADC、DAC
- **功能**：核心控制、通信管理

#### 1.2 电源模块
- **参数**：DC 5V/1A
- **接口**：Micro USB
- **功能**：为终端供电

### 2. 核心功能模块（仅保留必要硬件）

#### 2.1 继电器模块（4路）
- **型号**：4路继电器模块
- **参数**：
  - 控制电压：5V
  - 负载电压：250VAC/30VDC
  - 负载电流：10A
  - 隔离：光耦隔离
- **接口**：GPIO
- **功能**：客房设备控制（灯光、插座、窗帘等）- **核心功能**
- **说明**：硬件控制，确保断网时仍可本地控制

#### 2.2 RGB LED模块（状态指示）
- **型号**：WS2812B
- **参数**：
  - 数量：5颗
  - 控制方式：单线数字控制
  - 颜色：全彩RGB
- **接口**：GPIO
- **功能**：客房状态指示 - **核心功能**
- **说明**：硬件指示，实时反馈系统状态

#### 2.3 红外发射模块
- **型号**：红外发射管+驱动电路
- **参数**：
  - 载波频率：38kHz
  - 控制方式：GPIO脉冲控制
- **接口**：GPIO
- **功能**：空调/电视红外控制 - **核心功能**
- **说明**：支持红外学习和发射

#### 2.4 红外接收模块
- **型号**：VS1838B，38kHz
- **参数**：
  - 工作电压：2.7V-5.5V
  - 接收距离：15-20米
- **接口**：GPIO
- **功能**：红外遥控学习 - **核心功能**
- **说明**：支持红外码学习

#### 2.5 RFID模块
- **型号**：RC522
- **参数**：
  - 工作频率：13.56MHz
  - 支持卡片：Mifare S50、S70等
  - 读卡距离：≤5cm
  - 读卡时间：＜200ms
- **接口**：I2C
- **功能**：门禁读卡 - **核心功能**
- **说明**：支持房卡读取和门禁管理

#### 2.6 📞 语音通话模块（v2.0新增 - 双向）
- **组成**：
  - **INMP441 I2S数字麦克风**（¥8）：语音采集，16kHz采样率
  - **PAM8403功放 + 3W扬声器**（¥15）：语音播放，前台回复/广播/铃声
- **接口**：
  - 麦克风：I2S（WS/SD/SCK，复用RFID引脚需时分）
  - 扬声器：ESP32 DAC输出 → RC低通滤波 → PAM8403
- **功能**：与前台实时双向语音通信（住客发起/前台主动发起） / 一键呼叫 / 前台广播通知
- **说明**：ESP32双核架构下，Core0处理I/O+MQTT，Core1专责音频编解码

## 🎯 功能需求（"轻硬件+重软件"设计理念）

### 1. 核心硬件功能（必须保留）

#### 1.1 继电器控制
- **功能描述**：控制客房设备（灯光、插座、窗帘等）
- **功能要求**：
  - 支持4路继电器控制
  - 支持本地控制（断网可用）
  - 支持远程控制
  - 支持定时控制
- **说明**：硬件控制，确保可靠性

#### 1.2 状态指示
- **功能描述**：指示客房终端工作状态
- **功能要求**：
  - 正常工作：绿色LED
  - 通信异常：红色LED
  - 报警状态：闪烁红色LED
  - 配网状态：蓝色LED
- **说明**：硬件指示，实时反馈

#### 1.3 红外遥控
- **功能描述**：控制空调、电视等红外设备
- **功能要求**：
  - 支持红外学习
  - 支持红外发射
  - 支持多种品牌空调/电视
  - 支持本地和远程控制
- **说明**：硬件控制，无需云端

#### 1.4 RFID门禁
- **功能描述**：读取房卡，管理门禁
- **功能要求**：
  - 支持Mifare卡读取
  - 支持本地开门
  - 支持远程开门
  - 支持门禁记录
- **说明**：硬件控制，确保安全性

#### 1.5 📞 语音通话（v2.0新增 - 硬件+软件协同 - 双向）
- **功能描述**：客房端与前台之间的实时双向语音通信（住客发起/前台主动发起）
- **硬件要求**：
  - INMP441 I2S数字麦克风采集语音（16kHz/16bit）
  - PAM8403功放+3W扬声器播放前台语音
  - 独立"呼叫前台键"触发住客发起通话
  - 蜂鸣器响铃提示前台来电
- **功能要求**：
  - 一键呼叫前台，OLED显示呼叫状态
  - 前台来电时铃声提示 + LED闪烁
  - 双向实时音频传输（目标延迟≤200ms）
  - 住客发起：30秒无应答自动挂断
  - 前台发起：15秒无应答自动挂断
  - 通话中支持静音/免提切换
  - 与SOS功能联动：SOS时自动发起紧急通话
- **通信协议选型**：
  | 方案 | 优势 | 劣势 | 适用场景 |
  |------|------|------|----------|
  | **WebRTC**（推荐） | 低延迟、自适应码率、NAT穿透 | 需信令服务器 | 局域网/互联网 |
  | MQTT + Opus编码 | 简单、复用现有MQTT基础设施 | 延迟略高(~300ms) | 纯局域网 |
  | WebSocket + PCM | 最简单实现 | 无压缩、带宽占用大 | 调试/原型 |
- **说明**：ESP32双核架构下 Core1专责音频编解码，不影响Core0的设备控制

### 2. 软件功能（通过酒店APP实现）

#### 2.1 环境监测（软件实现）
- **功能描述**：实时监测客房环境
- **实现方式**：酒店APP集成环境监测数据
- **功能要求**：
  - 温湿度数据：通过云端API获取
  - 空气质量数据：通过云端API获取
  - 光照强度数据：通过云端API获取
  - 数据可视化展示
- **说明**：数据存储在云端，APP实时展示

#### 2.2 设备控制（软件实现）
- **功能描述**：通过APP控制客房设备
- **实现方式**：酒店APP + 云端控制
- **功能要求**：
  - 灯光控制：开关、亮度、颜色
  - 空调控制：温度、模式、定时
  - 窗帘控制：开关、位置
  - 电视控制：开关、频道
  - 场景模式：一键切换
- **说明**：软件控制，功能丰富

#### 2.3 门禁管理（软件实现）
- **功能描述**：通过APP实现门禁管理
- **实现方式**：酒店APP数字房卡
- **功能要求**：
  - 数字房卡：手机开锁
  - 远程开门：前台/APP控制
  - 门禁记录：云端存储
  - 门锁状态：云端查询
- **说明**：参考酒店APP数字房卡设计

#### 2.4 安防管理（软件实现）
- **功能描述**：通过APP实现安防管理
- **实现方式**：酒店APP + 云端监控
- **功能要求**：
  - 门窗状态：云端查询
  - 人体检测：云端查询
  - 报警响应：云端处理
  - 远程报警：APP推送
- **说明**：数据存储在云端，APP实时推送

#### 2.5 节能管理（软件实现）
- **功能描述**：通过APP实现节能管理
- **实现方式**：酒店APP + AI算法
- **功能要求**：
  - 无人自动断电：AI算法判断
  - 空调节能控制：AI算法优化
  - 灯光自动调节：AI算法控制
  - 能耗统计：云端报表
- **说明**：AI算法在云端，APP展示报表

#### 2.6 互动服务（软件实现）
- **功能描述**：通过APP实现互动服务
- **实现方式**：酒店APP AI智能体
- **功能要求**：
  - AI管家：问答、服务预约
  - 服务工单：自动生成
  - 物品配送：调度机器人
  - 酒店信息：在线查询
- **说明**：参考酒店APPAI管家设计

### 1. 酒店APP核心功能

#### 1.2 数字房卡
- **功能**：手机开锁，无需实体房卡
- **启示**：硬件无需RFID读卡器，APP生成数字房卡

#### 1.3 AI智能体管家
- **功能**：问答、服务预约、工单生成

#### 1.4 语音控制
- **功能**："Hello酒店"唤醒，控制设备
- **启示**：语音识别在云端，硬件仅采集音频

#### 1.5 无接触服务
- **功能**：自助入住、0秒退房、在线开票
- **启示**：流程在云端，硬件仅执行指令

### 2. "轻硬件+重软件"设计原则

#### 2.1 硬件简化
- **只保留核心功能**：继电器控制、红外遥控、RFID门禁、状态指示
- **移除复杂功能**：传感器、语音识别、摄像头
- **降低开发成本**：减少硬件开发时间

#### 2.2 软件增强
- **云端AI算法**：环境监测、节能优化、AI管家
- **APP交互界面**：地图选房、数字房卡、语音控制
- **数据可视化**：能耗统计、设备状态、报警信息

#### 2.3 成本控制
- **硬件成本**：¥90（标准版）
- **软件成本**：云端AI、APP开发
- **总成本**：低于传统方案
