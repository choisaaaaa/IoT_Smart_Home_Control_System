## 概述

### 功能说明
智能家居分控设备支持通过USB接口连接电脑，使用串口终端进行调试和配置。无需额外硬件，ESP32内置USB转串口功能。

### 主要功能
1. **WiFi配网**：通过串口快速配置WiFi连接
2. **传感器调试**：实时查看传感器数据和状态
3. **设备控制**：手动控制继电器、LED、电控锁等设备
4. **参数配置**：修改系统参数和配置
5. **故障排查**：查看系统日志，定位问题
6. **固件升级**：通过串口烧录新固件

### 优势
- **无需额外硬件**：ESP32内置USB转串口
- **操作简单**：使用串口终端即可
- **功能全面**：支持所有调试功能
- **成本低廉**：只需一根USB线
- **稳定可靠**：比蓝牙配网更稳定

## 硬件准备

### 必需硬件
1. **USB数据线**（1根）
   - 接口类型：Micro USB或Type-C（根据开发板接口选择）
   - 长度建议：1-2米
   - 要求：必须支持数据传输，不能是充电线

2. **电脑**（1台）
   - 操作系统：Windows 7/10/11、macOS、Linux
   - 接口：USB接口
   - 要求：能够安装串口驱动和调试软件

### 可选硬件
1. **USB集线器**（1个）
   - 用途：连接多个分控设备
   - 建议：使用带供电的USB集线器

2. **USB延长线**（1根）
   - 用途：延长USB连接距离
   - 长度：1-5米

## 软件准备

### 串口驱动安装

#### Windows系统
1. **下载驱动**
   - 访问ESP32官方驱动下载页面
   - 下载CP210x或CH340驱动（根据开发板型号选择）
   - 下载地址：https://www.silabs.com/developers/usb-to-uart-bridge-vcp-drivers

2. **安装驱动**
   - 双击下载的驱动安装包
   - 按照提示完成安装
   - 安装完成后重启电脑

3. **验证安装**
   - 打开"设备管理器"
   - 展开"端口(COM和LPT)"
   - 查看是否显示COM端口（如COM3、COM4等）

#### macOS系统
1. **下载驱动**
   - 访问ESP32官方驱动下载页面
   - 下载macOS版本的CP210x或CH340驱动

2. **安装驱动**
   - 双击下载的.dmg文件
   - 按照提示完成安装
   - 安装完成后重启电脑

3. **验证安装**
   - 打开终端
   - 输入命令：`ls /dev/cu.*`
   - 查看是否显示USB串口设备（如/dev/cu.usbserial-xxx）

#### Linux系统
1. **安装驱动**
   - 大多数Linux系统自带驱动，无需额外安装
   - 如需安装，使用系统包管理器安装

2. **验证安装**
   - 打开终端
   - 输入命令：`ls /dev/ttyUSB*`或`ls /dev/ttyACM*`
   - 查看是否显示USB串口设备

### 串口终端软件

#### 推荐软件

**1. Arduino IDE串口监视器**（免费，推荐新手）
- **优点**：免费、简单易用、与Arduino开发环境集成
- **缺点**：功能相对简单
- **下载地址**：https://www.arduino.cc/en/software

**2. PuTTY**（免费，推荐高级用户）
- **优点**：免费、功能强大、支持多种协议
- **缺点**：界面相对简陋
- **下载地址**：https://www.putty.org/

**3. SecureCRT**（收费，专业用户）
- **优点**：功能全面、界面友好、支持脚本
- **缺点**：收费软件
- **下载地址**：https://www.vandyke.com/products/securecrt/

**4. Tera Term**（免费，开源）
- **优点**：免费、开源、功能丰富
- **缺点**：界面较旧
- **下载地址**：https://ttssh2.osdn.jp/

**5. RealTerm**（免费，支持二进制）
- **优点**：免费、支持二进制数据、功能强大
- **缺点**：界面复杂
- **下载地址**：https://realterm.sourceforge.io/

## 连接方法

### 步骤1：连接硬件
1. 将USB数据线的一端连接到分控的Micro USB或Type-C接口
2. 将USB数据线的另一端连接到电脑的USB接口
3. 确保连接牢固，没有松动

### 步骤2：检查设备识别
1. 打开"设备管理器"（Windows）或终端（macOS/Linux）
2. 查看是否识别到新的COM端口或USB串口设备
3. 记下端口号（如COM3、/dev/cu.usbserial-xxx）

### 步骤3：打开串口终端
1. 打开选择的串口终端软件
2. 配置串口参数：
   - **波特率**：115200
   - **数据位**：8
   - **校验位**：None
   - **停止位**：1
   - **流控**：None
3. 选择正确的COM端口
4. 点击"打开"或"连接"按钮

### 步骤4：验证连接
1. 按下分控的复位按钮（如有）
2. 观察串口终端是否有输出信息
3. 如果看到启动信息，说明连接成功
4. 如果没有输出，检查连接和端口配置

## 常见问题

### 问题1：无法识别USB设备
**现象**：连接USB线后，电脑无法识别分控设备

**解决方案**：
1. 检查USB线是否为数据线（不是充电线）
2. 尝试更换USB线
3. 尝试更换USB接口
4. 重新安装USB驱动
5. 在其他电脑上测试

### 问题2：串口终端无输出
**现象**：连接成功，但串口终端没有输出信息

**解决方案**：
1. 检查串口参数配置是否正确（波特率115200）
2. 按下分控的复位按钮，重启设备
3. 检查COM端口是否选择正确
4. 尝试其他串口终端软件
5. 检查USB线连接是否牢固

### 问题3：WiFi配网失败
**现象**：输入WiFi信息后，连接失败

**解决方案**：
1. 检查WiFi密码是否正确
2. 检查WiFi频段是否支持（只支持2.4GHz）
3. 检查WiFi信号是否足够强（>-75dBm）
4. 尝试扫描WiFi网络，确认目标WiFi可见
5. 检查路由器是否允许新设备连接

### 问题4：传感器数据异常
**现象**：读取传感器数据时，数值异常或错误

**解决方案**：
1. 检查传感器连接是否正确
2. 检查传感器供电是否正常
3. 按下复位按钮，重启设备
4. 查看系统日志，定位问题
5. 如仍有问题，可能是传感器损坏

### 问题5：命令无响应
**现象**：输入命令后，设备无响应

**解决方案**：
1. 检查命令是否正确
2. 检查串口终端是否正确发送命令
3. 按下复位按钮，重启设备
4. 查看系统日志，检查是否有错误
5. 尝试使用其他串口终端软件

## 调试技巧

### 技巧1：使用日志功能
在调试过程中开启日志功能，可以查看详细的运行信息：
```
log on
```
这样可以更容易定位问题。

### 技巧2：定期保存配置
在配置完成后，建议定期保存配置，避免丢失：
```
info
```
查看系统信息，确认配置已保存。

### 技巧3：使用脚本自动化
对于重复性的调试任务，可以使用脚本自动化：
- PuTTY支持脚本功能
- SecureCRT支持Python脚本
- 可以编写脚本自动执行一系列命令

### 技巧4：记录调试过程
建议记录调试过程，包括：
- 使用的命令
- 输出结果
- 遇到的问题
- 解决方案
这样方便后续参考和问题排查。

### 技巧5：使用多个串口窗口
对于多个分控设备，可以打开多个串口窗口，同时调试多个设备。

### 技巧6：使用串口日志功能
将串口输出保存到文件，方便后续分析：
- PuTTY：Session Logging
- SecureCRT：Log Session
- Tera Term：Log
