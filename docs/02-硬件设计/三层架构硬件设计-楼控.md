# 智慧酒店物联网控制系统 - 三层架构硬件设计-楼层控制器

## 📋 文档概述

### 文档说明
本文档详细描述智慧酒店物联网控制系统的三层架构中**楼层控制器（原楼控）**的硬件设计，包括硬件组成、接线表、功能说明、演示功能等内容。

### 设备定位
楼层控制器是智慧酒店三层架构的**中间层核心设备**，负责：
- **楼层级集中管控**：管理整层楼的所有客房终端和公共区域设备
- **环境监测与联动**：监测公共区域环境，自动控制走廊灯光、空调等
- **客房状态汇总**：收集并显示该层所有客房的状态信息
- **🚔安全报警（公安合规）**：入侵检测增强、通行日志≥90天存储、防拆报警
- **🚒消防联动（GB 50116合规）**：烟雾探测、消防主机通信(RS485)、应急照明/疏散指示控制、防火卷帘联动
- **能耗统计**：汇总整层楼的能耗数据
- **访客通行管理**：楼层门禁控制

##### 2.6 温湿度传感器(DHT11) + 空气质量传感器(MQ-135) + 光照传感器 → 🌡️ 环境监测演示
- **功能**：公共区域环境三合一监测
- **演示场景**：
  - OLED实时滚动显示三项数值
  - 用手握住DHT11 → 温度上升 → 超过28℃自动开启继电器3（模拟空调）
  - 用酒精棉靠近MQ-135 → PPM值飙升 → RGB LED第8颗变红 + 蜂鸣器预警 + 开启继电器4（新风机）
  - 遮挡光敏电阻 → 光照降低 → 低于150lux自动开启继电器1（走廊灯）

##### 2.7 人体红外传感器(HC-SR501) + RGB灯带(WS2812B×6颗) → 💡 走廊智能照明演示
- **功能**：模拟走廊人来灯亮、人走灯灭
- **演示逻辑**：
  ```
  检测到有人进入 → RGB灯带渐亮(暖白,80%亮度)
  → 人在持续移动 → 保持常亮
  → 人离开后无人移动30秒 → 渐暗至关闭
  → 同时光照传感器判断：白天亮度40%，夜晚亮度100%
  ```
- **效果**：RGB灯带模拟走廊灯带，有人经过时亮起，离开后自动熄灭

##### 2.8 RFID模块(RC522) + 继电器(1路) → 🔐 楼层门禁演示
- **功能**：模拟楼层门禁刷卡进出
- **演示场景**：
  - 刷有效卡 → 蜂鸣器"滴"一声 → OLED显示"欢迎: 张三(3F)" → 继电器吸合模拟开门（绿灯亮）
  - 刷无效卡 → 蜂鸣器"滴滴"报警声 → OLED显示"❌ 无效卡" → 上报前台管理端（红灯闪烁）
  - 支持远程下发权限：前台管理端通过MQTT发送新卡号 → 楼层控制器自动添加白名单
  - 记录通行日志：时间+卡号+结果 → OLED报警页面可查看

##### 2.9 舵机(SG90) + 继电器(1路) → 🚒 消防联动演示
- **功能**：模拟火灾时的防火卷帘下降和排烟系统
- **演示场景**：
  - 按下按键2切换到"消防演练"模式 或 手动触发
  - **第一阶段（火灾确认）**：
    - 蜂鸣器连续长鸣（火灾警报声）
    - 全部RGB LED快速红闪
    - OLED显示"🚒 火灾警报！请立即疏散！"
    - MQTT广播通知所有客房端进入紧急模式
  - **第二阶段（联动执行）**：
    - 舵机从0°转到90° → 模拟**防火卷帘半降**
    - 3秒后转到180° → 模拟**防火卷帘全降**
    - 继电器2吸合 → 模拟**排烟风机启动**
    - 继电器1强制开启 → 模拟**应急照明（走廊灯全开）**
    - 继电器4吸合 → 模拟**新风机最大转速排烟**
  - **第三阶段（恢复）**：
    - 按键3确认解除 → 所有设备复位
    - 舵机回到0°，继电器释放，报警停止

##### 2.10 继电器模块（6路）→ ⚡ 公共区域设备控制演示 + 🚒消防联动执行
- **6路继电器分配**：

| 继电器 | 控制对象 | 触发条件 | 演示效果 |
|--------|----------|----------|----------|
| 继电器1 | 走廊灯光 | 人体感应+光照不足 | RGB灯带通电/断电模拟 |
| 继电器2 | 排烟风机 | 消防联动时启动 | 可接小风扇模拟 |
| 继电器3 | 电梯厅空调 | 温度超过28℃ | 可接LED表示运行 |
| 继电器4 | 新风系统 | 空气质量>50ppm | 可接小风扇模拟 |
| **继电器5** | **🚒应急照明** | **火灾时强制切换** | **常闭→消防时吸合，切换到应急电源** |
| **继电器6** | **🚒疏散指示** | **火灾时点亮** | **常开→消防时吸合，点亮疏散指示灯/安全出口标志** |

- **手动/自动双模式**：可通过按键切换，自动模式下由传感器驱动，手动模式下按键直接控制
- **消防联动优先级**：当收到火警信号时，继电器5/6不受模式限制，**立即执行**消防动作

##### 2.11 烟雾传感器(MQ-2) → 🚒 消防探测核心
- **功能**：走廊公共区域烟雾浓度检测
- **技术参数**：
  - 检测范围：300-10000ppm
  - 工作电压：5V（需预热）
  - 输出方式：模拟量(ADC) + 数字量(可调阈值)
  - 响应时间：<10秒
- **报警联动逻辑**：
  ```
  MQ-2 ADC值 > 阈值(如800)
    → 触发一级预警：蜂鸣器"滴"间隔3秒 + LED#8黄色闪烁
    → OLED显示"⚠️ 烟雾预警"
  
  MQ-2 ADC值 > 高阈值(如1500) 或 持续30秒超阈值
    → 触发二级火警：全套消防联动自动执行
    → 上报前台管理端(RS485+MQTT双重上报)
    → 舵机转动(卷帘下降)
    → 继电器5吸合(应急照明)
    → 继电器6吸合(疏散指示)
    → 继电器2吸合(排烟风机)
    → MQTT广播所有客房端进入紧急模式
  ```

## 🎯 硬件功能需求（楼层控制器）

### 1. 楼层集中管控功能

#### 1.1 客房状态汇总
- 通过MQTT收集该层所有客房端的状态（在线/离线、入住/空置、设备状态）
- OLED分页显示各客房状态
- RGB LED实时反映每个客房的状态变化
- 定时（每10秒）向前台管理端上报楼层汇总数据

#### 1.2 能耗统计
- 收集各客房端上报的能耗数据
- 计算楼层总能耗
- 支持按日/周/月统计（存储在Flash中）
- 上报前台管理端用于整体能耗分析

### 2. 环境监测与联动功能

#### 2.1 三合一环境监测
- 温度监测（DHT11）：范围0-50℃，精度±2℃
- 湿度监测（DHT11）：范围20-90%RH，精度±5%RH
- 空气质量监测（MQ-135）：范围10-1000ppm
- 光照强度监测（光敏电阻）：范围0-1000lux
- 采样频率：2秒/次

#### 2.2 智能联动规则
| 条件 | 动作 | 延迟 |
|------|------|------|
| 温度 > 28℃ | 开启继电器3（空调） | 即时 |
| 温度 < 20℃ | 关闭继电器3（空调） | 即时 |
| 空气质量 > 50ppm | 开启继电器4（新风机） | 即时 |
| 空气质量恢复 < 30ppm | 关闭继电器4（新风机） | 延迟60秒 |
| 光照 < 150lux 且 检测到人 | 开启继电器1（走廊灯） | 即时 |
| 无人 > 30秒 | 关闭继电器1（走廊灯） | 延迟30秒 |

### 3. 公共区域智能控制功能

#### 3.1 走廊智能照明
- 人体红外传感器检测人员存在
- 结合光照传感器实现白天低亮度、夜晚高亮度
- RGB灯带模拟走廊灯带，支持渐亮/渐灭过渡效果
- 支持手动覆盖（按键强制开关）

#### 3.2 楼层模式切换
| 模式 | 走廊灯 | 空调 | 新风 | 说明 |
|------|--------|------|------|------|
| 正常模式 | 自动感应 | 自动恒温 | 自动 | 默认运行模式 |
| 节能模式 | 关闭/低亮 | 调高2℃ | 降低风速 | 夜深或无客人时段 |
| 夜间模式 | 10%亮度(感应时) | 关闭 | 关闭 | 23:00-06:00 |
| 消防演练 | 强制全开 | 关闭 | 最大转速 | 演示用 |

### 4. 安全管理与报警功能

#### 4.1 楼层门禁
- RFID刷卡验证（RC522）
- 白名单存储在Flash中（最多50张卡）
- 支持前台管理端通过MQTT远程添加/删除卡片
- **通行日志记录（≥90天存储）**：卡号+时间+结果
- 无效卡尝试超过3次触发入侵警报

#### 4.2 入侵检测与防拆报警
- **非法闯入**：未授权人员触发门禁失败3次以上 → 蜂鸣器报警 + LED红闪 + MQTT上报前台
- **🚔防拆报警**（新增）：设备被从安装位置拆除时触发（可配合震动传感器或倾斜开关）
  - 触发条件：设备倾斜角度>30° 或 震动强度超阈值
  - 联动动作：蜂鸣器急促长鸣 + 全部LED红闪 + MQTT上报前台 + RS485上报安防系统
  - 复位方式：按键3确认 + 设备重新固定

#### 4.3 🚔楼层SOS报警（新增）
- **触发**：SOS键短按或长按
- **联动动作**：
  1. 蜂鸣器"滴—滴—"(两声长)提示
  2. OLED显示"🚔 SOS已发送! 等待前台响应..."
  3. MQTT消息 `hotel/floor/{floor_id}/sos` 上报前台管理端
  4. 前台收到后可通过MQTT下发指令解除或联动110
- **复位**：再次按SOS 或 前台远程解除

#### 4.4 🚔安全审计日志系统（≥90天）
- **法规依据**：《旅馆业治安管理办法》+ 公安部安全监控要求
- **日志类型与保留期**：

| 日志类型 | 数据字段 | 保留期 |
|----------|----------|--------|
| 门禁通行 | 卡号/时间/位置(楼层)/结果(通过/拒绝) | ≥90天 |
| 入侵事件 | 时间/来源(门禁/防拆)/严重级别 | ≥90天 |
| SOS报警 | 触发时间/来源/处理状态 | ≥90天 |
| 火警事件 | 时间/探测器读数(MQ-2值)/处理过程 | ≥90天 |
| 操作日志 | 操作者(按键/远程)/指令/时间 | ≥30天 |

- **存储介质**：ESP32 Flash分区（4MB中分配1MB给日志区，支持循环覆盖）
- **导出方式**：
  - USB串口CSV格式导出（`export_logs` 命令）
  - MQTT上传云端平台
- **演示场景**：串口输入 `export_logs` → 输出最近10条日志记录

### 5. 🚒 消防联动系统（符合GB 50116《火灾自动报警系统设计规范》）

#### 5.1 烟雾探测（MQ-2）
- **探测范围**：走廊公共区域（模拟整层烟雾监测）
- **工作模式**：持续ADC采样（每2秒一次），需预热30秒
- **阈值设定**：
  | 级别 | ADC阈值 | 含义 | 动作 |
  |------|---------|------|------|
  | 正常 | <800 | 无异常 | 正常运行 |
  | 一级预警 | 800-1500 | 轻度烟雾 | 黄色LED闪烁 + OLED提示 + 蜂鸣器间隔音 |
  | 二级火警 | >1500 或 持续>30s | 明显火情 | **全套消防联动自动执行** |
- **抗误报设计**：
  - 连续3次采样均超阈值才判定为有效报警
  - 支持灵敏度调节（通过OLED菜单调整阈值）

#### 5.2 消防主机通信（RS485/MODBUS）
- **通信接口**：RS485总线（MAX3485模块）
- **通信协议**：MODBUS RTU（从站模式），9600bps, 8N1
- **连接对象**：酒店消防火灾报警控制器（主机）
- **接收数据**（与前台管理端一致）：
  | 寄存器地址 | 数据含义 | 说明 |
  |------------|----------|------|
  | 0x0000 | 火警状态字 | bit0=火警 bit1=故障 bit2=屏蔽 bit3=监管 |
  | 0x0001 | 火警回路号 | 触发火警的探测器所在回路 |
  | 0x0002 | 火警地址码 | 触发火警的探测器地址 |
  | 0x0010 | 故障类型 | 0=正常 1=主电故障 2=备电故障 3=通讯故障 |
  | 0x0020 | 系统运行状态 | 0=正常 1=火警 2=故障 3=自检 4=屏蔽 |
- **心跳检测**：每30秒轮询消防主机状态
- **双重上报通道**：收到火警信号时同时通过RS485回传确认 + MQTT上报前台

#### 5.3 消防联动执行序列
```
火警触发（MQ-2超标 OR 消防主机信号 OR 前台疏散指令）
    │
    ├─→ [T+0s] 本地响应
    │     ├─ 蜂鸣器：连续长鸣（消防警报音）
    │     ├─ RGB LED：全部红闪（2Hz频率）
    │     └─ OLED："🚒 火灾警报！请立即疏散！"
    │
    ├─→ [T+1s] 继电器动作（消防优先级最高）
    │     ├─ 继电器2 吸合 → 排烟风机启动
    │     ├─ 继电器5 吸合 → 应急照明切换（常闭→吸合）
    │     └─ 继电器6 吸合 → 疏散指示点亮（常开→吸合）
    │
    ├─→ [T+2s] 卷帘控制
    │     └─ 舵机转动：0°→45°(半降,延迟10s)→90°(全降)
    │         （模拟防火卷帘两段式下降）
    │
    ├─→ [T+3s] 远程通知
    │     ├─ MQTT广播 `hotel/emergency/evacuate`
    │     │   → 所有客房端进入紧急模式
    │     │   → 前台管理端显示火警位置
    │     └─ RS485 MODBUS回传确认到消防主机
    │
    └─→ [持续] 循环执行直到复位
          └─ 复位条件：按键3手动复位 OR 消防主机复位信号
```

#### 5.4 备用电源接口预留
- **目的**：火灾时市电可能断电，楼控需继续运行至少30分钟（GB 25506要求）
- **接口设计**：电源模块预留DC 12V备用电池接入端子
- **切换逻辑**：市电正常时充电；市电中断时无缝切换到电池供电
- **演示方案**：标注接口位置，实际部署时接UPS或锂电池组

#### 5.5 多级报警体系（整合公安/消防）
| 报警级别 | 触发条件 | 声音 | LED | 上报 | 联动动作 |
|----------|----------|------|-----|------|----------|
| 提示级 | 环境参数接近阈值 | 短滴(3s间隔) | 黄色闪烁 | 是 | 无 |
| 警告级 | 环境参数超限/客房呼叫 | 双滴(1s间隔) | 橙色闪烁 | 是 | 自动调节设备 |
| 🚔紧急级 | SOS/非法闯入/防拆 | 快速连击 | 红色快闪 | 是+广播 | 通知前台管理端+记录日志≥90天 |
| 🚒灾难级 | **火灾(MQ-2/消防主机)** | 连续长鸣(消防音) | 红色全闪 | 是+广播全楼 | **执行消防联动全套(卷帘+排烟+应急照明+疏散指示)** |

```
4. 使用未录入的卡刷 → "滴滴"报警 → OLED显示"❌ 无效卡"
5. 连续刷3次无效卡 → 触发入侵警报模式
```

### 场景四：消防联动演示（最震撼）
```
1. 按下按键2切换到"消防演练"模式（或长按按键4）
2. 【第1秒】蜂鸣器开始连续长鸣（火灾警报）
3. 【第1秒】全部RGB LED快速红闪
4. 【第1秒】OLED显示"🚒 火灾警报！请疏散！"
5. 【第3秒】舵机转到90°（防火卷帘半降）
6. 【第6秒】舵机转到180°（防火卷帘全降）
7. 【第3秒】继电器2吸合（排烟风机启动，听到"咔哒"）
8. 【第3秒】继电器1吸合（应急照明全开）
9. 【第3秒】继电器4吸合（新风机最大转速）
10. 整个过程语音解说配合
11. 按键3 → 全部复位，恢复正常
```

### 场景五：跨设备联动演示（与前台管理端/客房端配合）
```
前提：前台管理端和至少一台客房端同时运行

1. 前台管理端下发"301退房指令" → 楼层控制器接收 → 转发给301客房端
2. 301客房端执行退房流程（关灯/关空调/关门锁）
3. 301客房端上报完成 → 楼层控制器更新OLED显示（301变为⚪空置）
4. 301客房端触发SOS紧急呼叫 → 楼层控制器收到 → 蜂鸣器报警
5. 楼层控制器转发给前台管理端 → 前台管理端OLED弹出警告
6. 前台管理端回复"已处理" → 楼层控制器停止报警
```

## 🔧 硬件调试功能

### 1. USB串口调试
- 波特率：115200
- 日志内容：传感器原始值、继电器状态、MQTT消息、报警事件
- 支持命令调试：
  - `set_temp 35` - 模拟温度值
  - `trigger_fire` - 触发消防演练
  - `add_card ABC12345678` - 添加门禁卡
  - `list_rooms` - 显示客房状态

### 2. WiFi配网
- SmartConfig一键配网
- AP模式配网页面（手机连接后配置）
- 配网信息存Flash，掉电不丢失
- 配网状态：RGB LED第1颗蓝闪=配网中，绿=成功

### 3. OTA固件升级
- 支持通过WiFi无线升级固件
- 升级过程中RGB LED黄闪
- 升级失败自动回滚到上一版本
