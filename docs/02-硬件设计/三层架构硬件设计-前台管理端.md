# 智慧酒店物联网控制系统 - 三层架构硬件设计-前台管理端（电脑外设版）

## 📋 文档概述

### 文档说明
本文档详细描述智慧酒店物联网控制系统中**前台管理端**的硬件设计。前台管理端作为**前台电脑的外设附件**，通过USB连接前台电脑，负责物理层交互（房卡读写、状态可视化、紧急报警），而显示、语音、网络通信、数据存储等由电脑软件承担。

### 设备定位
前台管理端是部署于酒店前台的**电脑外设附件**，定位为"轻硬件+重软件"：

```
┌─────────────────────────────────────────────────────┐
│                  🏨 前台工作台                       │
│                                                     │
│  ┌──────────┐         ┌─────────────────────────┐   │
│  │ 前台电脑  │◄──USB──►│  前台管理端(本设备)      │   │
│  │          │  串口    │                         │   │
│  │ ·显示器   │         │ ·RFID房卡读写(RC522)    │   │
│  │ ·麦克风   │         │ ·RGB状态墙LED(×8颗)     │   │
│  │ ·扬声器   │         │ ·SOS报警键              │   │
│  │ ·网络     │         │ ·蜂鸣器                │   │
│  │ ·AI软件   │         │ ·RS485消防接口          │   │
│  │ ·数据库   │         │ ·UART公安接口           │   │
│  └──────────┘         └─────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 2. 核心硬件模块详解

##### 2.1 主控板（ESP32-S2-MINI 或 STM32F103C8T6）
- **选型理由**：
  - ESP32-S2：原生USB OTG支持，可作为USB CDC Device（虚拟串口），开发简单
  - STM32F103C8T6：更廉价，USB FS Device，成熟稳定
- **工作模式**：USB从机（Device）模式，模拟CDC串口设备
- **与电脑通信协议**：自定义文本帧协议（见下文通信协议章节）
- **供电方式**：USB 5V取电，板载LDO降压至3.3V

##### 2.2 RFID模块(RC522) → 💳 房卡读写
- **功能**：Mifare Classic 1K卡的读写操作
- **操作类型**：
  | 指令 | 说明 | 触发方式 |
  |------|------|----------|
  | `CARD_WRITE` | 写入房卡（房号/有效期/权限） | 电脑下发写卡指令+数据 |
  | `CARD_READ` | 读取房卡信息 | 检测到卡片自动读取并上报 |
  | `CARD_INVALIDATE` | 注销房卡（退房时） | 电脑下发注销指令 |
  | `CARD_BLACKLIST` | 加入黑名单（挂失） | 电脑下发黑名单指令 |
- **演示场景**：

**发卡流程**
```
电脑端GUI: 用户点击"办理301入住"
    ↓ USB发送
本设备: 收到 WRITE_CARD {room:"301", expire:"2026-04-05", level:"guest"}
    ↓ 等待放卡
用户放置空白卡 → RC522检测到卡片
    ↓
本设备: 写入Sector → 成功/失败
    ↓ USB上报
电脑端: 收到 CARD_RESULT {success:true, uid:"ABCD1234"}
    ↓
电脑端: GUI显示"✅ 房卡制作成功!" + 更新数据库 + MQTT通知楼控
```

**读卡流程**
```
用户将房卡放在RC522感应区
    ↓
本设备: 自动读取 → 解析数据
    ↓ USB上报
电脑端: 收到 CARD_DATA {room:"301", guest:"张三", checkin:"04-04 14:00", valid:true}
    ↓
电脑端: GUI显示住客详情 + 匹配当前业务（入住/退房/查询）
```

##### 2.3 RGB灯带(WS2812B×8颗) → 🏨 客房状态可视化墙
- **功能**：物理展示全楼层客房实时状态
- **驱动方式**：电脑通过USB下发颜色指令，本设备驱动WS2812B
- **颜色编码**：

| 颜色 | 含义 | 触发条件 |
|------|------|----------|
| 🟢 绿色 | 已入住 | 电脑收到楼控上报"已入住"→下发SET_LED指令 |
| ⚪ 白色(灭) | 空置 | 默认状态/退房完成后 |
| 🔴 红色 | 维修中 | 前台标记维修 |
| 🟡 黄色 | 清洁中 | 退房→清洁中过渡态（持续5秒→变白） |
| 🔵 蓝色闪烁(2Hz) | SOS报警 | 收到客房/楼控SOS上报 |
| 🟣 紫色 | 已预订 | 前台预订但未入住 |

- **USB指令格式**：
  ```
  SET_LED <index> <color>       // 设置单颗LED颜色
  SET_LED_ALL <c0> <c1> ... <c7> // 批量设置全部8颗
  LED_FLASH <index> <color> <hz> // 闪烁效果（用于报警）
  LED_BREATH <index> <color>     // 呼吸效果（用于入住动画）
  ```

##### 2.4 按键模块 + SOS报警键
- **按键分配**：

| 按键 | 名称 | 上报指令 | 说明 |
|------|------|----------|------|
| 按键1 | 入住办理 | `BTN_CHECKIN` | 通知电脑进入入住模式 |
| 按键2 | 退房办理 | `BTN_CHECKOUT` | 通知电脑进入退房模式 |
| 按键3 | 模式切换 | `BTN_MODE` | 标准↔节能↔紧急模式循环 |
| 按键4 | 报警确认 | `BTN_ACK` | 确认当前报警 |
| **SOS键** | **110报警** | **`BTN_SOS`** | **紧急报警，长按3秒增强** |

- **SOS报警联动动作**（硬件层执行）：
  1. 蜂鸣器播放警笛音效（三长短循环）
  2. 全部RGB LED红闪（系统4颗+状态墙8颗）
  3. USB立即上报 `BTN_SOS` 给电脑
  4. 电脑收到后：GUI弹窗+音箱播放报警语音+MQTT广播全楼警戒

##### 2.5 蜂鸣器（有源蜂鸣器，5V）
- **音效定义**（硬件层直接播放，无需电脑参与）：

| 场景 | 音效 | 触发方式 |
|------|------|----------|
| 房卡写入成功 | "滴—"(单声长) | 本地RC522写卡成功 |
| 房卡读取成功 | "滴"(单声短) | 本地RC522读到有效卡 |
| 操作错误 | "滴滴"(双声短) | 无效卡/写卡失败 |
| **🚔110报警触发** | **三长短急促循环** | **SOS键按下（硬件直驱）** |
| **🚒消防警报** | **连续长鸣+间歇停顿** | **电脑下发 `BUZZER_FIRE` 指令** |

##### 2.6 RS485模块(MAX3485) → 🚒 消防主机接口
- **功能**：透传式RS485通信桥接
- **工作模式**：电脑 ↔ USB ↔ 本设备(RS485) ↔ 消防主机
- **协议**：MODBUS RTU，9600bps, 8N1
- **数据流向**：
  ```
  电脑发送: RS485_SEND <hex_data>     → 本设备转RS485发出
  电脑接收: RS485_RECV <hex_data>     ← 本设备从RS485收到后上报
  ```
- **消防主机寄存器定义**（同楼控端一致）：

| 寄存器地址 | 数据含义 | 说明 |
|------------|----------|------|
| 0x0000 | 火警状态字 | bit0=火警 bit1=故障 bit2=屏蔽 bit3=监管 |
| 0x0001 | 火警回路号 | 触发火警的探测器所在回路 |
| 0x0002 | 火警地址码 | 触发火警的探测器地址 |
| 0x0010 | 故障类型 | 0=正常 1=主电故障 2=备电故障 3=通讯故障 |
| 0x0020 | 系统运行状态 | 0=正常 1=火警 2=故障 3=自检 4=屏蔽 |

##### 2.7 UART扩展口(4Pin) → 🚔 公安接口预留
- **功能**：预留身份证读卡器/公安终端连接口
- **引脚定义**：
  ```
  Pin1: VCC (5V)
  Pin2: GND
  Pin3: TXD (本设备→外设)
  Pin4: RXD (本设备←外设)
  ```
- **工作模式**：透传模式，电脑通过USB下发/接收UART数据
- **可对接设备**：
  - 公安身份证读卡器（SAM模块）
  - 公安实名登记终端
  - 110报警联网设备

##### 2.8 RGB LED状态指示(WS2812B×4颗)
- **设备自身状态指示**（独立于状态墙）：

| LED | 含义 | 颜色 |
|-----|------|------|
| LED1 | USB连接状态 | 绿色=已连接 / 红色=断开 |
| LED2 | 设备就绪 | 绿色=正常 / 黄色=忙碌 |
| LED3 | 业务状态 | 蓝=入住中 / 橙=退房中 / 红=报警中 |
| LED4 | 消防/公安状态 | 绿=正常 / 红闪烁=火警/SOS |

### 总电流估算
- **3.3V负载**：~186mA
- **5V负载**：~540mA
- **USB 2.0供电能力**：500mA（标准）/ 900mA（充电端口）
- **建议**：使用USB 3.0端口或带供电Hub（确保5V/1A以上）

## 📡 USB通信协议

### 协议帧格式
```
<帧头> <指令> <参数> <校验> <帧尾>

帧头:   0xAA 0x55 (固定2字节)
指令:   1字节 ASCII命令码
参数:   变长 JSON字符串
校验:   1字节 XOR校验
帧尾:   0x0D 0x0A (\r\n)
```

### 指令集

#### 电脑 → 本设备（下行指令）

| 指令码 | 指令名 | 参数 | 说明 |
|--------|--------|------|------|
| `W` | CARD_WRITE | `{room,expire,level}` | 写入房卡 |
| `I` | CARD_INVALIDATE | `{uid}` | 注销房卡 |
| `B` | CARD_BLACKLIST | `{uid}` | 加入黑名单 |
| `L` | SET_LED | `{index,color}` | 设置状态墙单颗LED |
| `LA` | SET_LED_ALL | `[c0,c1,...,c7]` | 批量设置全部LED |
| `LF` | LED_FLASH | `{index,color,hz}` | LED闪烁（报警用） |
| `LB` | LED_BREATH | `{index,color}` | LED呼吸（入住动画） |
| `Z` | BUZZER | `{sound_id}` | 播放指定音效 |
| `ZF` | BUZZER_FIRE | — | 消防警报音效 |
| `R` | RS485_SEND | `{hex_data}` | 发送RS485数据 |
| `U` | UART_SEND | `{data}` | 发送UART数据 |
| `G` | GET_STATUS | — | 查询设备状态 |
| `E` | ECHO | `{text}` | 测试回显 |

#### 本设备 → 电脑（上行上报）

| 上报码 | 上报名 | 参数 | 触发条件 |
|--------|--------|------|----------|
| `CD` | CARD_DATA | `{uid,room,guest,checkin,valid,...}` | 检测到房卡自动读取 |
| `CR` | CARD_RESULT | `{success,uid,error?}` | 写卡/注销操作结果 |
| `BN` | BTN_PRESS | `{btn_id}` | 按键被按下 |
| `BS` | BTN_SOS | `{duration}` | SOS键按下（含长按时长） |
| `RR` | RS485_RECV | `{hex_data}` | RS485收到数据 |
| `UR` | UART_RECV | `{data}` | UART收到数据 |
| `ST` | STATUS | `{usb,rfid,leds,...}` | 状态查询响应 |
| `ER` | ERROR | `{code,msg}` | 错误上报 |

### 3. 🚔 公安对接（电脑软件 + 硬件接口）

#### 3.1 实名登记
- 方案A（推荐）：UART扩展口连接身份证读卡器 → 电脑通过USB透传读取
- 方案B：电脑直接连接USB身份证读卡器（绕过本设备）
- 读卡数据 → 本地存储 → MQTT上传云端 → 转发公安平台

#### 3.2 110一键报警
- SOS键硬件直驱（蜂鸣器+LED，不依赖电脑）
- 同时USB上报 `BTN_SOS` 给电脑
- 电脑软件响应：
  1. GUI最高级别弹窗
  2. 电脑音箱播放预录报警语音
  3. MQTT广播全楼警戒
  4. 可选：通过调制解调器/VoIP拨打110
  5. 记录完整事件日志（时间/持续时间/操作人）

#### 3.3 异常人员预警
- 黑名单数据库存储在电脑SQLite
- 入住时自动比对姓名/证件号
- 命中则：GUI红色告警 + 状态墙对应LED紫闪 + 蜂鸣器急促提示

### 4. 🚒 消防对接（电脑软件 + RS485硬件接口）

#### 4.1 消防主机监控
- 电脑定时（每30秒）通过本设备RS485轮询消防主机状态
- 解析MODBUS寄存器获取火警/故障/运行状态
- GUI实时显示消防主机状态面板

#### 4.2 火警响应
- 收到火警信号后电脑软件自动执行：
  1. GUI红色全屏警告
  2. 电脑音箱循环播放疏散指令（TTS或预录音频）
  3. 下发 `BUZZER_FIRE` 让本设备蜂鸣器也响
  4. 下发 `LED_FLASH` 让状态墙全部红闪
  5. MQTT广播 `hotel/emergency/evacuate` 到所有楼控和客房端
  6. 记录火警事件日志

#### 4.3 手动消防演练
- GUI按钮"启动消防演练" → 同上全套流程
- 用于日常演练和实物演示

### 详细演示脚本

#### 场景一：入住办理演示（最核心🏨）
```
1. 电脑端点击"演示-入住办理" 或 说"开始入住演示"
2. 电脑TTS："请将空白房卡放在读卡区"
3. GUI显示: "⏳ 等待放卡..."
4. 【第1步】用户放置空白Mifare卡
   → 本设备RC522检测到 → 蜂鸣器"滴" → USB上报 CARD_DATA(空卡)
   → GUI显示: "✓ 检测到空白卡，正在写入..."
5. 【第2步】电脑自动下发写卡指令
   → RC522写入: room=301, expire=明天12:00, level=guest
   → 约2秒 → 写入成功
   → 蜂鸣器"滴—"（长声确认）
   → USB上报: CARD_RESULT {success:true}
6. 【第3步】电脑端响应
   → GUI大字: "✅ 301房卡制作成功!"
   → 状态墙LED#2(对应301): 绿色呼吸动画
   → 电脑TTS: "房卡已制作完成！301房间，祝您入住愉快"
   → (后台)MQTT通知楼控更新房态
7. 完成！整个流程约10-15秒
```

#### 场景二：🚔公安110报警演示
```
1. 演示员按下SOS键
   → 【硬件层-即时】蜂鸣器警笛音效(三长短循环)
   → 【硬件层-即时】全部LED(4+8颗)红闪
   → 【硬件层-即时】USB上报 BTN_SOS {duration:0}
2. 电脑端收到:
   → GUI弹出最大级警告窗: "🚔 110报警已触发!"
   → 电脑音箱播放: "这里是XX酒店前台，发生紧急情况，请求支援"
   → 状态墙保持红闪
3. (可选) 演示员长按SOS超过3秒
   → 上报 BTN_SOS {duration:3500}
   → 电脑端显示: "长按确认! 正在尝试拨通110..."
4. 再次按SOS 或 点击GUI"解除" → 全部复位
```

#### 场景三：🚒消防联动演示
```
1. 启动方式（三选一）:
   A. 电脑GUI点击"🚒 启动消防演练"
   B. 电脑串口输入: sim_fire
   C. (如有真实消防主机) 触发火警
2. 【T+0s】电脑端:
   → GUI红色警告: "🚒 火灾警报! 请立即疏散!"
   → 电脑音箱播放疏散指令TTS
   → 下发 BUZZER_FIRE → 本设备蜂鸣器消防警报音
   → 下发 LED_FLASH 全部红闪
3. 【T+1s】电脑端网络:
   → MQTT广播: hotel/emergency/evacuate
   → (如楼控/客房端在线) → 楼控卷帘下降+排烟启动; 客房端开锁亮灯关空调
4. 【持续】循环播放直到复位
5. 复位: GUI点击"解除" → 全部恢复正常
```
