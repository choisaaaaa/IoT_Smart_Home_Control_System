# 智慧酒店物联网控制系统 - 三层架构前台管理端需求

## 📋 文档概述

### 文档说明
本文档描述智慧酒店物联网控制系统的三层架构中前台管理端硬件需求，包括硬件组成、功能需求、接口定义、调试功能等。

### 设备定位
前台管理端是智慧酒店三层架构的前台核心设备，部署于酒店前台，负责入住退房办理、房卡管理、客房状态监控、AI前台助理、指令协调、应急指挥等功能。

---

## 🔧 硬件组成

### 1. 核心控制模块

#### 1.1 ESP32主控板
- **型号**：ESP32-WROOM-32
- **参数**：
  - CPU：双核Xtensa LX6，240MHz
  - Flash：4MB
  - RAM：520KB
  - 无线：WiFi 802.11 b/g/n、蓝牙4.2 BLE
  - 接口：GPIO、UART、I2C、SPI、ADC、DAC
- **功能**：系统核心控制，数据处理，通信管理

#### 1.2 电源模块
- **参数**：DC 5V/2A
- **接口**：Micro USB
- **功能**：为主控和所有外设供电
- **稳压电路**：AMS1117-3.3V（3.3V稳压）

### 2. 人机交互模块

#### 2.1 显示模块
- **型号**：0.96寸OLED显示屏
- **分辨率**：128×64
- **接口**：I2C
- **功能**：显示客房列表、入住信息、系统状态、操作界面

#### 2.2 输入模块
- **按键模块**：4键独立按键
  - 按键1：入住办理
  - 按键2：退房办理
  - 按键3：模式切换（标准/节能/紧急）
  - 按键4：报警确认

#### 2.3 输出反馈模块
- **蜂鸣器**：有源蜂鸣器，5V供电（操作提示音、报警声、房卡操作反馈）
- **RGB LED(系统状态)**：WS2812B×4颗（系统/网络/业务/消息状态指示）

### 3. 房卡管理与状态显示模块

#### 3.1 RFID模块
- **型号**：RC522，13.56MHz
- **接口**：I2C/SPI
- **功能**：房卡读卡写卡（Mifare Classic 1K）
- **支持卡片**：Mifare S50白卡

#### 3.2 客房状态可视化墙
- **型号**：WS2812B×8颗灯带
- **接口**：单总线GPIO
- **功能**：8颗LED分别代表8间客房的实时状态
- **颜色编码**：
  - 白色 = 空置可入
  - 绿色 = 已入住
  - 黄色 = 清洁中
  - 红色 = 维修中
  - 紫色闪烁 = SOS报警

### 4. AI前台助理模块

#### 4.1 麦克风模块
- **型号**：INMP441
- **接口**：I2S数字麦克风
- **功能**：语音采集，AI前台助理输入

#### 4.2 扬声器模块
- **型号**：3W 8Ω扬声器 + 功放芯片(PAM8403)
- **接口**：DAC/PWM
- **功能**：语音播放，AI前台助理输出

### 5. 连接材料
- **面包板**：400孔，带电源总线
- **杜邦线套装**：公对公、公对母、母对母，60根/套
- **排针排母**：2.54mm间距，40P
- **电源线**：DC 5.5×2.1mm接口

### 6. 外壳与结构
- **塑料外壳**：透明塑料盒，150×100×50mm
- **固定螺丝**：M3×10mm，4颗

## 🎯 功能需求

### 1. 入住退房管理功能

#### 1.1 功能描述
RFID房卡读写与入住退房流程管理

#### 1.2 功能要求
- 支持Mifare Classic 1K卡片的读写操作
- 写入房间号、有效期、权限等级到房卡
- 发卡成功后自动通知楼控更新房态
- 支持房卡挂失、续期、换房操作
- 蜂鸣器提供操作成功/失败的声音反馈
- OLED显示当前操作步骤和结果

### 2. 客房状态监控功能

#### 2.1 功能描述
全酒店客房状态一览与可视化

#### 2.2 功能要求
- OLED分页显示所有客房状态（已入住/空置/维修/清洁中）
- RGB灯带实时同步每间房的颜色状态
- 实时计算并显示入住率
- 接收并处理来自楼控/客房端的报警信息
- 报警时对应LED闪烁+蜂鸣器报警

### 3. AI前台助理功能

#### 3.1 功能描述
语音交互式前台服务

#### 3.2 功能要求
- 麦克风采集语音信号
- 本地语音识别（支持关键词唤醒）
- 语音引导式入住/退房办理流程
- 客房状态查询、设备控制指令
- 扬声器TTS语音反馈
- 多轮对话支持

### 4. 指令协调中心功能

#### 4.1 功能描述
三层架构的指令协调与数据汇总枢纽

#### 4.2 功能要求
- 下发给指定客房终端的控制指令
- 下发给指定楼控的管理指令
- 全楼模式切换（节能模式/欢迎模式/紧急模式）
- 收集各终端数据并上报云端
- 接收来自各终端的报警信息并处理

### 5. 应急指挥功能

#### 5.1 功能描述
紧急事件处理与消防联动触发

#### 5.2 功能要求
- 一键进入紧急模式（按键或语音）
- 触发消防联动（通知楼控执行全套联动动作）
- 全楼广播紧急消息
- 所有LED进入红色闪烁状态
- 报警确认后恢复正常

## 🔌 硬件接口定义

### 主控板引脚定义

| ESP32引脚 | 功能 | 连接设备 | 说明 |
|-----------|------|----------|------|
| 3.3V | 电源输出 | 所有3.3V设备 | OLED/RFID/麦克风供电 |
| GND | 接地 | 所有设备接地 | 公共地 |
| 5V | 电源输入 | 5V电源模块 | 输入5V电源 |
| GPIO21 | I2C SDA | OLED显示屏 | I2C数据线 |
| GPIO22 | I2C SCL | OLED显示屏 | I2C时钟线 |
| GPIO25 | I2C SDA | RFID模块(RC522) | I2C数据线 |
| GPIO26 | I2C SCL | RFID模块(RC522) | I2C时钟线 |
| GPIO15 | GPIO | 按键1 | 入住办理 |
| GPIO2 | GPIO | 按键2 | 退房办理 |
| GPIO0 | GPIO | 按键3 | 模式切换 |
| GPIO4 | GPIO | 按键4 | 报警确认 |
| GPIO16 | GPIO | 蜂鸣器 | 声音反馈 |
| GPIO5 | GPIO | RGB LED(系统状态) | WS2812B×4系统状态指示 |
| GPIO18 | GPIO | RGB灯带(状态墙) | WS2812B×8客房状态可视化 |
| GPIO33 | I2S SD | 麦克风(INMP441) | 语音采集数据 |
| GPIO27 | I2S WS | 麦克风(INMP441) | 语音采集时钟 |
| GPIO14 | I2S SCK | 麦克风(INMP441) | 语音采集位时钟 |
| GPIO12 | DAC/PWM | 扬声器功放(PAM8403) | 语音播放 |
| GPIO13 | UART TX | USB转TTL | 调试串口 |
| GPIO3 | UART RX | USB转TTL | 调试串口 |

## 🔧 调试功能

### 1. USB串口调试
- **功能描述**：通过USB串口进行调试
- **功能要求**：
  - 串口波特率：115200
  - 支持日志输出
  - 支持命令调试
  - 支持固件升级

### 2. WiFi配网
- **功能描述**：通过WiFi进行配网
- **功能要求**：
  - 支持热点模式配网
  - 支持STA模式配网
  - 支持配网状态显示
  - 支持配网日志记录

### 3. RFID调试
- **功能描述**：RFID模块调试与测试
- **功能要求**：
  - 支持读取卡片UID
  - 支持写入测试数据
  - 支持扇区格式化
  - 显示读写结果
