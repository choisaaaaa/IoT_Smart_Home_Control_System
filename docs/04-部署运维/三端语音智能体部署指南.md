## 🎯 比赛部署建议

### 为什么推荐云服务器？

在比赛场景下，使用云服务器具有以下优势：

#### ✅ 优势

| 优势 | 说明 |
|------|------|
| **环境一致性** | 所有评审人员看到的都是相同配置的环境 |
| **快速部署** | 一键脚本即可完成部署，方便演示 |
| **性能保障** | 云服务器性能稳定，避免本地环境差异 |
| **远程演示** | 可以通过URL直接分享给评委，无需本地安装 |
| **易于审核** | 源码+部署脚本即可复现完整环境 |

#### 💡 推荐方案

**方案1: 学生云服务器（最经济）**
- 阿里云/腾讯云学生优惠：9.9元/月
- 配置：1核2GB，足够演示使用
- 优势：成本低，性能足够

**方案2: 本地开发 + Docker打包**
- 在本地开发完成后，使用Docker打包
- 提供 `docker-compose.yml` 和部署脚本
- 评审人员只需运行一条命令即可部署

**方案3: 混合方案（推荐）**
- 本地开发使用 Docker（环境隔离）
- 比赛演示使用云服务器（远程访问）
- 提供完整的部署文档和脚本

## 🚀 比赛快速部署指南

### 方案1: 本地开发 + Docker部署（推荐）

**适合场景**: 开发阶段，环境隔离，易于调试

```bash
# 1. 克隆项目
git clone https://github.com/your-repo/smart-home.git
cd smart-home

# 2. 配置环境变量
cd backend
cp .env.example .env
# 编辑 .env 文件，填写 API Keys

# 3. 启动所有服务
cd ..
docker-compose up -d

# 4. 查看服务状态
docker-compose ps

# 5. 访问后端 API
curl http://localhost:3000/api/v1/health
```

**适合场景**: 比赛演示，远程访问，环境一致

```bash

curl -fsSL https://get.docker.com -o get-docker.sh

curl ifconfig.me
```

**适合场景**: 快速部署，简化流程

./deploy.sh

#### 方式2: Docker 部署（推荐）

##### 创建 `Dockerfile`

```dockerfile
# 使用官方 Node.js 18 镜像
FROM node:18-alpine

# 设置工作目录
WORKDIR /app

# 安装构建依赖（可选，用于原生模块）
RUN apk add --no-cache python3 make g++

# 复制 package 文件
COPY package*.json ./

# 安装依赖
RUN npm ci --only=production && npm cache clean --force

# 复制源代码
COPY . .

# 暴露端口
EXPOSE 3000

# 设置环境变量
ENV NODE_ENV=production

# 启动应用
CMD ["npm", "start"]
```

##### 创建 `docker-compose.yml`

```yaml
version: '3.8'

services:
  # 后端服务
  backend:
    build: .
    container_name: smart-home-backend
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DB_HOST=db
      - REDIS_HOST=redis
      - MQTT_HOST=mqtt
    depends_on:
      - db
      - redis
      - mqtt
    restart: unless-stopped
    volumes:
      - ./logs:/app/logs
      - ./config:/app/config

  # MySQL 数据库
  db:
    image: mysql:8.0
    container_name: smart-home-db
    ports:
      - "3306:3306"
    environment:
      MYSQL_ROOT_PASSWORD: your_root_password
      MYSQL_DATABASE: smart_home
      MYSQL_USER: smart_home_user
      MYSQL_PASSWORD: your_password
    volumes:
      - mysql_data:/var/lib/mysql
    restart: unless-stopped

  # Redis 缓存
  redis:
    image: redis:7-alpine
    container_name: smart-home-redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped

  # MQTT Broker
  mqtt:
    image: eclipse-mosquitto:2.0
    container_name: smart-home-mqtt
    ports:
      - "1883:1883"
      - "9001:9001"  # WebSocket
    volumes:
      - mosquitto_data:/mosquitto
    restart: unless-stopped

volumes:
  mysql_data:
  redis_data:
  mosquitto_data:
```

##### 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f backend

# 停止服务
docker-compose down

# 重启服务
docker-compose restart
```

### 2.6 Nginx 反向代理（生产环境）

#### 安装 Nginx

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install nginx

# CentOS/RHEL
sudo yum install nginx
```

#### 配置 Nginx

```nginx
# /etc/nginx/sites-available/smart-home
server {
    listen 80;
    server_name your-domain.com;  # 或服务器IP

    # 前端静态文件
    location / {
        root /var/www/smart-home-frontend;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # WebSocket 支持
        proxy_read_timeout 86400;
    }

    # 音频文件代理
    location /audio/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_cache off;
    }

    # 日志
    access_log /var/log/nginx/smart-home-access.log;
    error_log /var/log/nginx/smart-home-error.log;
}
```

#### 启用配置

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/smart-home /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx

# 设置开机自启
sudo systemctl enable nginx
```

### 8.2 API 认证

后端已实现 JWT 认证，前端需要在请求头中携带 Token:

```javascript
// src/api/index.js
import axios from 'axios';

const api = axios.create({
  baseURL: 'http://localhost:3000/api/v1',
  timeout: 10000
});

// 请求拦截器
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器
api.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      // Token 过期，跳转登录
      localStorage.removeItem('token');
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default api;
```
