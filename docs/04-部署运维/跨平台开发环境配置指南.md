# 跨平台开发环境配置指南

## 📋 文档说明

本文档详细说明如何在Windows本地开发环境和Ubuntu云服务器环境之间进行开发协作，确保开发环境一致性，提高团队协作效率。

**文档版本**: v1.0.0  
**最后更新**: 2026年4月4日  
**适用项目**: IoT Smart Hotel 智能酒店控制系统

---

## 🎯 核心原则

### 1. 环境一致性

**问题**: Windows本地开发，Ubuntu云服务器部署，环境差异导致"在我机器上能运行"

**解决方案**: 使用Docker容器化开发，确保环境完全一致

```
┌─────────────────────────────────────────────────────────────┐
│                    开发环境一致性方案                         │
├─────────────────────────────────────────────────────────────┤
│  Windows本地开发    →    Docker容器    →    Ubuntu云服务器    │
│  (开发调试)          (开发环境)          (生产部署)            │
│      ↓                    ↓                    ↓              │
│   VS Code           VS Code / CLI         Docker / PM2      │
│   + Docker          + Docker              + Nginx             │
└─────────────────────────────────────────────────────────────┘
```

### 2. 开发流程

```
┌─────────────────────────────────────────────────────────────┐
│                    开发流程                                   │
├─────────────────────────────────────────────────────────────┤
│  1. 本地Windows开发 (Docker)                                 │
│     ↓                                                        │
│  2. 本地测试验证                                             │
│     ↓                                                        │
│  3. 提交代码到Git                                            │
│     ↓                                                        │
│  4. 云服务器Ubuntu拉取代码                                    │
│     ↓                                                        │
│  5. Docker部署到生产环境                                     │
└─────────────────────────────────────────────────────────────┘
```

## 🖥️ Windows本地开发环境配置

### 方案1: Docker Desktop + VS Code (推荐)

#### 1. 安装Docker Desktop

**下载地址**: https://www.docker.com/products/docker-desktop/

**安装步骤**:
1. 下载Docker Desktop安装包
2. 运行安装程序
3. 启动Docker Desktop
4. 验证安装: `docker --version`

#### 2. 安装VS Code

**下载地址**: https://code.visualstudio.com/

**推荐插件**:
- Docker
- Remote - Containers
- GitLens
- ESLint
- Prettier
- Python
- C/C++

#### 3. 克隆项目

```bash
# 在PowerShell或CMD中执行
git clone https://github.com/your-username/iot-smart-hotel.git
cd iot-smart-hotel
```

#### 4. 启动开发环境

```bash
# 启动所有服务（MySQL, Redis, MQTT等）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f
```

#### 5. 本地开发

**后端开发**:
```bash
# 进入后端目录
cd backend

# 安装依赖
npm install

# 启动开发服务器（热重载）
npm run dev

# 访问: http://localhost:3000
```

**前端开发**:
```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 访问: http://localhost:8080
```

### 方案2: WSL2 + Ubuntu (高级用户)

#### 1. 启用WSL2

```powershell
# 以管理员身份运行PowerShell
wsl --install
```

#### 2. 安装Ubuntu

```powershell
wsl --install -d Ubuntu-22.04
```

#### 3. 配置开发环境

```bash
# 在WSL2 Ubuntu中
# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
```

#### 4. VS Code远程开发

1. 安装"Remote - WSL"插件
2. 使用`WSL: New Window`打开WSL环境
3. 在WSL中打开项目目录

## 🌩️ Ubuntu云服务器环境配置

### 方案1: Docker部署 (推荐)

#### 1. 服务器选购建议

**配置推荐**:
- CPU: 2核
- 内存: 4GB
- 存储: 50GB SSD
- 带宽: 5Mbps
- 系统: Ubuntu 22.04 LTS

**推荐平台**:
- 阿里云学生机: 9.9元/月
- 腾讯云学生机: 9.9元/月
- 华为云学生机: 9.9元/月

#### 2. 服务器初始化

```bash
# SSH登录服务器
ssh root@your-server-ip

# 更新系统
apt-get update && apt-get upgrade -y

# 安装必要工具
apt-get install -y curl git wget
```

#### 3. 安装Docker

```bash
# 添加Docker官方GPG密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加Docker仓库
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 更新包索引
apt-get update

# 安装Docker
apt-get install -y docker-ce docker-ce-cli containerd.io

# 添加当前用户到docker组
usermod -aG docker $USER

# 验证安装
docker --version
```

#### 4. 安装Docker Compose

```bash
# 下载Docker Compose
curl -SL https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-linux-x86_64 -o /usr/local/bin/docker-compose

# 添加执行权限
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

#### 5. 部署项目

```bash
# 克隆项目
git clone https://github.com/your-username/iot-smart-hotel.git
cd iot-smart-hotel

# 配置环境变量
cd backend
cp .env.example .env
# 编辑.env文件，填写API Keys

# 返回项目根目录
cd ..

# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f
```

### 方案2: 原生部署 (不推荐)

```bash
# 安装Node.js
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
apt-get install -y nodejs

# 安装MySQL
apt-get install -y mysql-server
mysql_secure_installation

# 安装Redis
apt-get install -y redis-server

# 安装MQTT Broker
apt-get install -y mosquitto mosquitto-clients

# 配置PM2
npm install -g pm2

# 启动应用
cd backend
npm install
pm2 start pm2.config.js
```

## 🔄 环境同步方案

### 方案1: Docker Compose同步 (推荐)

**优势**: 环境完全一致，零配置部署

**本地开发**:
```bash
# Windows本地
docker-compose up -d
```

**云服务器部署**:
```bash
# Ubuntu服务器
docker-compose up -d
```

**环境变量同步**:
```bash
# 本地
cp backend/.env.example backend/.env
# 编辑本地.env

# 服务器
scp backend/.env user@server:/path/to/project/backend/
```

### 方案2: Git同步

**开发流程**:
```bash
# 本地开发
git add .
git commit -m "feature: 添加新功能"
git push origin main

# 服务器部署
git pull origin main
docker-compose down
docker-compose up -d
```

### 方案3: CI/CD自动化部署

**GitHub Actions配置**:
```yaml
name: Deploy to Ubuntu Server

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Deploy via SSH
      uses: appleboy/ssh-action@v1.0.0
      with:
        host: ${{ secrets.SERVER_HOST }}
        username: ${{ secrets.SERVER_USER }}
        key: ${{ secrets.SERVER_SSH_KEY }}
        port: ${{ secrets.SERVER_PORT }}
        script: |
          cd /path/to/project
          git pull origin main
          docker-compose down
          docker-compose up -d
```

## 🐛 常见问题解决

### 问题1: 端口冲突

**Windows本地**:
```bash
# 查看端口占用
netstat -ano | findstr :3000

# 结束进程
taskkill /PID <pid> /F
```

**Ubuntu服务器**:
```bash
# 查看端口占用
lsof -i :3000

# 结束进程
kill -9 <pid>
```

### 问题2: 文件权限问题

**Ubuntu服务器**:
```bash
# 修改目录权限
chmod -R 755 /path/to/project

# 修改文件所有者
chown -R $USER:$USER /path/to/project
```

### 问题3: Docker容器无法访问

**检查容器状态**:
```bash
docker-compose ps
docker-compose logs
```

**重启容器**:
```bash
docker-compose restart
docker-compose up -d
```

### 问题4: 网络配置问题

**Docker网络**:
```bash
# 查看网络
docker network ls

# 创建自定义网络
docker network create iot-network
```

## 🎯 最佳实践

### 1. 环境配置管理

**使用.env文件**:
```bash
# 统一配置模板
cp .env.example .env

# 本地开发: .env.local
# 服务器部署: .env.production
```

### 2. 版本控制

**提交规范**:
```
feat: 添加新功能
fix: 修复bug
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试相关
chore: 构建过程或辅助工具变动
```

### 3. 分支管理

```
main        # 生产环境
develop     # 开发环境
feature/*   # 功能分支
bugfix/*    # 修复分支
release/*   # 发布分支
```

### 4. 数据持久化

**Docker Compose配置**:
```yaml
volumes:
  mysql_data:
  redis_data:
  mqtt_data:
```

### 5. 日志管理

**本地开发**:
```bash
docker-compose logs -f
```

**服务器部署**:
```bash
# 配置日志轮转
docker-compose logs --tail=100 > logs/app.log
```
