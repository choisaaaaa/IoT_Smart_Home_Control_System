## 📊 数据库表结构

### 1. 设备表（devices）

**表说明**：存储所有设备信息

**表结构**：

| 字段名 | 类型 | 长度 | 是否主键 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|----------|--------|------|
| id | INT | 11 | 是 | 否 | - | 自增主键 |
| device_id | VARCHAR | 50 | 否 | 否 | - | 设备唯一标识 |
| device_type | VARCHAR | 20 | 否 | 否 | - | 设备类型 |
| device_name | VARCHAR | 50 | 否 | 否 | - | 设备名称 |
| device_key | VARCHAR | 50 | 否 | 否 | - | 设备密钥 |
| device_status | VARCHAR | 20 | 否 | 否 | 'offline' | 设备状态 |
| firmware_version | VARCHAR | 20 | 是 | 是 | NULL | 固件版本 |
| last_seen | DATETIME | - | 是 | 是 | NULL | 最后上线时间 |
| created_at | DATETIME | - | 否 | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | DATETIME | - | 否 | 否 | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引设计**：
- PRIMARY KEY (id)
- UNIQUE KEY (device_id)
- INDEX idx_device_id (device_id)
- INDEX idx_device_status (device_status)

**设备类型枚举**：
- `main` - 主控中心
- `sub1` - 分控1（环境监测）
- `sub2` - 分控2（安防管理）

**设备状态枚举**：
- `online` - 在线
- `offline` - 离线
- `error` - 异常

**示例数据**：
```sql
INSERT INTO devices (device_id, device_type, device_name, device_key, device_status, firmware_version) 
VALUES ('MAIN001', 'main', '主控中心', 'key_main_001', 'online', 'v1.0.0');
```

### 2. 传感器数据表（sensor_data）

**表说明**：存储传感器采集的数据

**表结构**：

| 字段名 | 类型 | 长度 | 是否主键 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|----------|--------|------|
| id | INT | 11 | 是 | 否 | - | 自增主键 |
| device_id | VARCHAR | 50 | 否 | 否 | - | 设备ID |
| sensor_type | VARCHAR | 20 | 否 | 否 | - | 传感器类型 |
| sensor_value | VARCHAR | 50 | 否 | 否 | - | 传感器值 |
| created_at | DATETIME | - | 否 | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引设计**：
- PRIMARY KEY (id)
- INDEX idx_device_id (device_id)
- INDEX idx_sensor_type (sensor_type)
- INDEX idx_created_at (created_at)
- INDEX idx_device_sensor (device_id, sensor_type)

**传感器类型枚举**：
- `temperature` - 温度
- `humidity` - 湿度
- `air_quality` - 空气质量
- `light_intensity` - 光照强度
- `door_status` - 门磁状态
- `motion` - 人体检测

**示例数据**：
```sql
INSERT INTO sensor_data (device_id, sensor_type, sensor_value)

```

### 3. 控制指令表（control_commands）

**表说明**：存储设备控制指令

**表结构**：

| 字段名 | 类型 | 长度 | 是否主键 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|----------|--------|------|
| id | INT | 11 | 是 | 否 | - | 自增主键 |
| device_id | VARCHAR | 50 | 否 | 否 | - | 设备ID |
| command_type | VARCHAR | 20 | 否 | 否 | - | 指令类型 |
| command_value | VARCHAR | 50 | 否 | 否 | - | 指令值 |
| command_status | VARCHAR | 20 | 否 | 否 | 'pending' | 指令状态 |
| created_by | VARCHAR | 50 | 是 | 是 | NULL | 创建者 |
| created_at | DATETIME | - | 否 | 否 | CURRENT_TIMESTAMP | 创建时间 |

- PRIMARY KEY (id)
- INDEX idx_device_id (device_id)
- INDEX idx_created_at (created_at)

- `lock_close` - 上锁

- `relay_off` - 继电器关

**指令状态枚举**：
- `pending` - 待执行
- `executing` - 执行中
- `success` - 成功
- `failed` - 失败
- `timeout` - 超时

**示例数据**：
```sql
INSERT INTO control_commands (device_id, command_type, command_value, command_status, created_by) 
VALUES ('SUB1001', 'light_on', '1', 'pending', 'user001');
```

### 5. 安防事件表（security_events）

**表说明**：存储安防相关事件

**表结构**：

| 字段名 | 类型 | 长度 | 是否主键 | 是否为空 | 默认值 | 说明 |
|--------|------|------|----------|----------|--------|------|
| id | INT | 11 | 是 | 否 | - | 自增主键 |
| device_id | VARCHAR | 50 | 否 | 否 | - | 设备ID |
| event_type | VARCHAR | 20 | 否 | 否 | - | 事件类型 |
| event_data | TEXT | - | 是 | 是 | NULL | 事件数据 |
| event_level | VARCHAR | 20 | 否 | 否 | 'info' | 事件级别 |
| created_at | DATETIME | - | 否 | 否 | CURRENT_TIMESTAMP | 创建时间 |

**索引设计**：
- PRIMARY KEY (id)
- INDEX idx_device_id (device_id)

**事件类型枚举**：

- `door_broken` - 门磁被破坏
- `card_swipe` - 刷卡开门

**事件级别枚举**：
- `info` - 信息
- `warning` - 警告
- `critical` - 严重
- `emergency` - 紧急

**示例数据**：
```sql
INSERT INTO security_events (device_id, event_type, event_data, event_level) 
VALUES ('SUB2001', 'intrusion', '{"sensor": "motion", "location": "living_room"}', 'critical');
```

## 📝 数据库索引优化

### 常用查询索引

#### 1. 设备状态查询
```sql
CREATE INDEX idx_devices_status ON devices(device_status);
```

#### 2. 传感器数据查询
```sql
CREATE INDEX idx_sensor_data_device_type ON sensor_data(device_id, sensor_type);
CREATE INDEX idx_sensor_data_time ON sensor_data(created_at);
```

#### 3. 控制指令查询
```sql
CREATE INDEX idx_commands_status ON control_commands(command_status);
CREATE INDEX idx_commands_device ON control_commands(device_id);
```

#### 4. 安防事件查询
```sql
CREATE INDEX idx_security_events_device ON security_events(device_id);
CREATE INDEX idx_security_events_time ON security_events(created_at);
```

#### 5. 系统日志查询
```sql
CREATE INDEX idx_logs_user ON system_logs(user_id);
CREATE INDEX idx_logs_time ON system_logs(created_at);
```

## 🔐 数据库安全

### 用户权限
```sql
-- 创建数据库用户
CREATE USER 'iot_user'@'localhost' IDENTIFIED BY 'password';

-- 授权
GRANT SELECT, INSERT, UPDATE, DELETE ON iot_system.* TO 'iot_user'@'localhost';
GRANT EXECUTE ON iot_system.* TO 'iot_user'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;
```

### 数据加密
- 密码字段使用bcrypt加密
- 敏感数据使用AES加密
- 传输使用SSL/TLS加密

## 📊 数据库备份

### 备份脚本
```bash
#!/bin/bash

# 备份数据库
mysqldump -u root -p<password> iot_system > /backup/iot_system_$(date +%Y%m%d).sql

# 压缩备份文件
gzip /backup/iot_system_$(date +%Y%m%d).sql

# 删除7天前的备份
find /backup -name "iot_system_*.sql.gz" -mtime +7 -delete
```

### 恢复脚本
```bash
#!/bin/bash

# 恢复数据库
gunzip -c /backup/iot_system_20240118.sql.gz | mysql -u root -p<password> iot_system
```
