# 智慧酒店物联网控制系统 - 软件子系统架构与开发路径

## 📋 文档概述

本文档汇总智慧酒店物联网控制系统的所有软件子系统，包括每个子系统的功能定位、技术架构、开发步骤、依赖关系和优先级，帮助团队快速理解系统全貌并制定开发计划。

**核心定位**：物联网设备控制与状态管理为核心，移除用户预订房间功能，聚焦设备控制、环境监测、客房服务等物联网核心场景。

---

## 🏗️ 一、后端服务子系统（9个）

### 1.1 核心框架子系统

#### 1.1.1 Express.js Web服务器
**功能定位**：后端HTTP服务入口，处理所有RESTful API请求

**技术架构**：
```
Node.js 20.x + Express.js 4.18 + TypeScript 5.4
├── 中间件层：CORS、日志、错误处理、JWT认证
├── 路由层：v1/ 下各模块路由
├── 控制器层：业务逻辑协调
└── 服务层：具体业务实现
```

**开发步骤**：
1. 项目初始化（npm init + TypeScript配置）
2. 安装依赖（express、typescript、cors、helmet等）
3. 创建基础架构（app.ts、server.ts、middleware/）
4. 配置CORS、日志、错误处理中间件
5. 实现JWT认证中间件
6. 创建路由分发（v1/）

**依赖关系**：无（最底层）
**优先级**：⭐⭐⭐⭐⭐（必须最先完成）
**预计开发时间**：1天

### 9.2 技术风险应对

| 风险 | 应对措施 |
|------|----------|
| **MQTT通信不稳定** | 1. 使用QoS 1确保消息可靠<br>2. 实现断线重连机制<br>3. 添加本地缓存 |
| **WebSocket实时性差** | 1. 使用Redis Adapter支持多实例<br>2. 优化消息广播策略<br>3. 减少不必要的消息 |
| **AI管家响应慢** | 1. 使用GLM-4-Flash（速度快）<br>2. 实现本地缓存<br>3. 添加超时重试 |
| **前端性能问题** | 1. 使用Vue 3 Composition API<br>2. 实现虚拟滚动<br>3. 优化ECharts图表渲染 |
| **固件稳定性问题** | 1. 使用FreeRTOS任务隔离<br>2. 实现看门狗机制<br>3. 添加断网本地控制 |
