# 网络配置功能说明

## 📡 网络配置概述

### 设计目标
1. **无需重新烧录**：所有网络参数可通过Web界面或App在线修改
2. **多设备支持**：主控和分控都支持在线配置
3. **配置持久化**：配置参数保存在Flash中，断电不丢失
4. **安全可靠**：配置界面需要密码保护，防止未授权访问
5. **用户友好**：提供直观的配置界面和操作指引

---

## 主控中心网络配置

### 1. WiFi配置

#### 配置方式

##### 方式一：Web配置界面（推荐）
**操作步骤：**
1. 主控启动时，如果检测到WiFi配置无效，自动进入AP模式
2. 主控创建WiFi热点：`SmartHome_Config`（密码：12345678）
3. 用户手机连接该热点
4. 浏览器访问：`192.168.4.1`
5. 进入WiFi配置页面
6. 选择或输入WiFi SSID和密码
7. 点击"保存并连接"
8. 主控重启，尝试连接目标WiFi
9. 连接成功后，热点自动关闭

**配置界面示例：**
```
┌─────────────────────────────────┐
│  SmartHome WiFi配置              │
├─────────────────────────────────┤
│  WiFi名称：[________________]    │
│  WiFi密码：[________________]    │
│  IP模式：  ○ DHCP  ● 静态IP     │
│  静态IP：  [192.168.1.100]     │
│  子网掩码：[255.255.255.0]     │
│  网关：    [192.168.1.1]        │
│  DNS：     [8.8.8.8]            │
│                                  │
│  [保存并连接]  [取消]            │
└─────────────────────────────────┘
```

##### 方式二：手机App配置
**操作步骤：**
1. 手机App启动，进入"设备配置"页面
2. 选择"WiFi配置"
3. App通过蓝牙或WiFi Direct与主控通信
4. 输入WiFi SSID和密码
5. 点击"发送配置"
6. 主控接收配置并尝试连接
7. 连接结果反馈到App

##### 方式三：按键+显示屏配置（离线配置）
**操作步骤：**
1. 按键4（系统重置）长按3秒，进入配置模式
2. OLED显示：`配置模式-WiFi`
3. 按键1/2切换配置项（SSID/密码）
4. 旋转编码器调节参数值
5. 按键3（确认）保存配置
6. OLED显示：`配置已保存，重启中...`
7. 主控重启，应用新配置

**配置流程：**
```
步骤1：选择SSID
- 按键1/2：扫描附近WiFi网络
- 旋转编码器：选择WiFi网络
- 按键3：确认选择

步骤2：输入密码
- 按键1/2：切换到密码输入
- 旋转编码器：选择字符（0-9, a-z）
- 按键3：确认当前字符
- 长按按键3：完成输入

步骤3：配置静态IP（可选）
- 按键1/2：切换到IP配置
- 旋转编码器：设置IP地址参数
- 按键3：确认配置
```

#### 配置参数说明

| 参数名称 | 说明 | 示例 | 备注 |
|----------|------|------|------|
| **WiFi SSID** | WiFi网络名称 | "MyWiFi_5G" | 必填 |
| **WiFi密码** | WiFi网络密码 | "password123" | 必填，8-63字符 |
| **IP模式** | IP地址获取方式 | DHCP/静态 | 可选，默认DHCP |
| **静态IP** | 设备静态IP地址 | 192.168.1.100 | 静态IP模式必填 |
| **子网掩码** | 网络子网掩码 | 255.255.255.0 | 静态IP模式必填 |
| **网关** | 网络网关地址 | 192.168.1.1 | 静态IP模式必填 |
| **DNS** | DNS服务器地址 | 8.8.8.8 | 可选，默认使用网关 |

### 3. 分控网络配置

#### 配置方式

分控的网络配置相对简单，主要是连接到主控的WiFi热点或局域网。

##### 方式一：自动连接主控热点
**操作步骤：**
1. 分控启动时，自动扫描附近WiFi网络
2. 查找名称为`SmartHome_Master_xxxx`的热点（xxxx为主控ID后4位）
3. 自动连接到主控热点（密码：12345678）
4. 连接成功后，获取IP地址
5. 向主控注册设备信息

##### 方式二：按键配置（连接局域网）
**操作步骤：**
1. 分控按键长按3秒，进入配置模式
2. 选择"连接局域网"模式
3. 通过按键输入或旋转编码器选择WiFi网络
4. 输入WiFi密码
5. 保存配置并重启
6. 分控连接到局域网
7. 通过mDNS或固定IP找到主控并注册

#### 配置参数说明

| 参数名称 | 说明 | 示例 | 备注 |
|----------|------|------|------|
| **连接模式** | 连接方式 | 主控热点/局域网 | 必填，默认主控热点 |
| **WiFi SSID** | 局域网SSID | MyWiFi_5G | 局域网模式必填 |
| **WiFi密码** | 局域网密码 | password123 | 局域网模式必填 |
| **主控IP** | 主控设备IP地址 | 192.168.1.100 | 局域网模式必填 |

### 2. 配置管理

#### 配置读取
- **启动时读取**：设备启动时从Flash读取配置
- **运行时读取**：运行时可通过API读取配置
- **配置验证**：读取后验证配置有效性

#### 配置写入
- **Web配置**：通过Web界面修改配置
- **App配置**：通过手机App修改配置
- **按键配置**：通过按键修改配置
- **远程配置**：通过云端下发配置（可选）

#### 配置备份与恢复
- **备份**：配置可导出为JSON文件
- **恢复**：从JSON文件恢复配置
- **云端备份**：配置自动备份到云端（可选）

### 3. 配置安全

#### 访问控制
- **Web界面密码**：访问Web配置界面需要输入密码（默认：admin）
- **App认证**：修改配置需要管理员权限
- **配置加密**：敏感配置（如密码）加密存储

#### 配置保护
- **防篡改**：配置文件签名验证
- **防误操作**：重要配置修改需要确认
- **配置锁定**：设置后可锁定配置，防止修改

## 故障排除

### 1. WiFi连接失败

#### 可能原因
1. WiFi密码错误
2. WiFi信号弱
3. 路由器设置问题
4. ESP32 WiFi模块故障

#### 解决方案
1. 检查WiFi密码是否正确
2. 靠近路由器，增强信号
3. 检查路由器是否允许新设备连接
4. 重启ESP32和路由器
5. 尝试连接其他WiFi网络

### 2. MQTT连接失败

#### 可能原因
1. MQTT服务器地址错误
2. 网络连接问题
3. MQTT服务未运行
4. 认证信息错误

#### 解决方案
1. 检查MQTT服务器地址和端口
2. 检查网络连接是否正常
3. 确认MQTT服务是否运行
4. 检查用户名和密码
5. 使用MQTT工具测试连接

### 3. 配置无法保存

#### 可能原因
1. Flash存储空间不足
2. Flash存储损坏
3. 配置格式错误

#### 解决方案
1. 清理Flash存储空间
2. 格式化Flash存储
3. 检查配置格式是否正确
4. 恢复出厂设置

### 4. 分控无法连接主控

#### 可能原因
1. 主控WiFi热点未开启
2. 分控WiFi模块故障
3. 距离过远，信号弱

#### 解决方案
1. 检查主控是否开启WiFi热点
2. 重启分控WiFi模块
3. 靠近主控，增强信号
4. 检查主控热点密码

## 开发实现

### 主控Web服务器实现

```cpp
#include <WiFi.h>
#include <WebServer.h>
#include <Preferences.h>

WebServer server(80);
Preferences preferences;

void setup() {
  Serial.begin(115200);
  
  // 初始化Preferences
  preferences.begin("config", false);
  
  // 检查WiFi配置
  String ssid = preferences.getString("wifi_ssid", "");
  String password = preferences.getString("wifi_password", "");
  
  if (ssid.length() == 0) {
    // 进入AP模式
    WiFi.softAP("SmartHome_Config", "12345678");
    Serial.println("AP模式启动");
    Serial.print("IP地址: ");
    Serial.println(WiFi.softAPIP());
    
    // 启动Web服务器
    setupWebServer();
  } else {
    // 连接WiFi
    WiFi.begin(ssid.c_str(), password.c_str());
    while (WiFi.status() != WL_CONNECTED) {
      delay(500);
      Serial.print(".");
    }
    Serial.println("WiFi连接成功");
    Serial.print("IP地址: ");
    Serial.println(WiFi.localIP());
  }
}

void setupWebServer() {
  // WiFi配置页面
  server.on("/", handleRoot);
  server.on("/config", handleConfig);
  server.on("/save", handleSave);
  
  server.begin();
}

void handleRoot() {
  server.send(200, "text/html", getWiFiConfigPage());
}

void handleConfig() {
  String ssid = server.arg("ssid");
  String password = server.arg("password");
  String mode = server.arg("mode");
  
  // 保存配置
  preferences.putString("wifi_ssid", ssid);
  preferences.putString("wifi_password", password);
  preferences.putString("wifi_mode", mode);
  
  server.send(200, "text/html", "<h1>配置已保存，设备将重启...</h1>");
  
  delay(1000);
  ESP.restart();
}

void loop() {
  server.handleClient();
}
```

## 总结

### 核心特点
1. **无需重新烧录**：所有网络参数可在线修改
2. **多种配置方式**：Web界面、App、按键三种方式
3. **配置持久化**：保存在Flash中，断电不丢失
4. **安全可靠**：密码保护，配置加密
5. **用户友好**：直观的界面和操作指引

### 适用场景
1. **首次配置**：设备首次使用时配置网络
2. **网络更换**：更换WiFi网络时重新配置
3. **服务器更换**：更换云端服务器时重新配置
4. **故障恢复**：网络故障时重新配置
5. **批量部署**：批量设备部署时统一配置

### 注意事项
