# 智联酒店 - 协同开发指南

> 本文档为团队成员协作开发提供统一规范，确保多人并行开发时代码风格一致、流程清晰、冲突可控。

## 一、项目架构总览

```
IoT_Smart_Hotel_System/
├── backend/iot-hotel-backend/    ← 后端服务（Node.js + Express + TS）
│   ├── src/
│   │   ├── controllers/          # 控制层（路由处理）
│   │   ├── services/             # 业务逻辑层
│   │   ├── models/               # 数据模型
│   │   ├── routes/               # 路由定义
│   │   ├── middleware/           # 中间件（JWT、错误处理等）
│   │   └── config/               # 配置（DB/MQTT/Redis）
│   ├── database/init.sql         # 数据库初始化脚本
│   ├── docker-compose.yml        # 本地 Docker 服务（MySQL+MQTT）
│   └── .env.example              # 环境变量模板
│
├── frontend/iot-hotel-web/       ← 前端应用（Vue3 + Vite + TS）
│   ├── src/
│   │   ├── api/                  # API 请求封装
│   │   ├── components/layout/    # 三套布局组件
│   │   ├── views/admin/          # 管理端页面（5个）
│   │   ├── views/reception/      # 前台端页面（7个）
│   │   ├── views/guest/          # 客户端页面（3个）
│   │   ├── stores/               # Pinia 状态管理
│   │   ├── types/                # TypeScript 类型定义
│   │   ├── router/               # Vue Router 路由配置
│   │   └── utils/                # 工具函数
│   └── vite.config.ts            # Vite 构建配置
│
├── docs/                         # 项目文档
├── docker/                       # Docker 部署配置
└── hardware/                     # 硬件固件代码
```

## 二、三层前端架构说明

| 层级 | 路由前缀 | 职责 | 页面数 |
|------|----------|------|--------|
| **管理端** | `/admin` | 设备监控、信息编辑、账单报表 | Dashboard / DeviceMonitor / RoomEdit / HotelInfoEdit / AdminReports |
| **前台端** | `/reception` | 入住退房、工单、送物、预订、余量、报表 | Dashboard / CheckInOut / Bookings / RoomAvailability / WorkOrders / DeliveryOrders / Bills |
| **客户端** | `/guest` | 预订入住、AI管家、送物、联系前台 | Booking / OnlineCheckIn / GuestRoom |

### 各端布局组件对应关系

- `AdminLayout.vue` → `/admin/*` 所有子路由
- `ReceptionLayout.vue` → `/reception/*` 所有子路由
- `GuestLayout.vue` → `/guest/*` 所有子路由

## 三、编码规范

### 3.1 统一格式（.editorconfig 已配置）

所有编辑器安装 **EditorConfig 插件**后自动生效：

| 规则 | 值 |
|------|-----|
| 字符集 | UTF-8 |
| 换行符 | LF（Unix风格） |
| 缩进 | 2空格 |
| 文件末尾 | 自动换行 |
| 去除行尾空格 | 是 |

### 3.2 TypeScript / Vue 规范

```typescript
// ✅ 正确：使用 function export，避免 default export 导致的 import 问题
export async function getRoomList(params?: any) { ... }

// ❌ 避免：default export 容易与动态导入冲突
// export default { getRoomList }

// ✅ 正确：Vue SFC 使用 <script setup lang="ts">
<script setup lang="ts">
import { ref, onMounted } from 'vue'
</script>
```

### 3.3 命名约定

| 类型 | 规范 | 示例 |
|------|------|------|
| **文件名** | PascalCase (组件) / camelCase (工具) | `AdminLayout.vue`, `request.ts` |
| **组件名** | PascalCase | `<DeviceMonitor />` |
| **变量/函数** | camelCase | `fetchRooms()`, `hotelStore` |
| **常量** | UPPER_SNAKE_CASE | `MAX_GUESTS` |
| **类型接口** | PascalCase + 后缀 | `RoomInfo`, `ApiResponse<T>` |
| **CSS类名** | kebab-case + BEM | `.device-card__status-online` |

### 3.4 Git 提交信息格式

```
<type>(<scope>): <subject>

type: feat | fix | docs | style | refactor | chore
scope: backend | frontend | admin | reception | guest | config | docs

示例：
feat(reception): 添加客房余量平面图视图
fix(guest): 修复在线办理入住步骤跳转问题
docs: 更新API接口文档 v2.0.0
chore: 完善.gitignore和.editorconfig
```

## 四、分支策略

```
main (受保护，仅接受PR)
  ├─ develop (日常开发主分支)
  │   ├─ feature/admin-reports     ← 管理端报表功能
  │   ├─ feature/guest-butler       ← 客户端AI管家
  │   ├─ feature/reception-checkin  ← 前台端入住流程
  │   └─ fix/room-availability-bug  ← 紧急修复
  │
  └─ release/v1.0.0                ← 发版分支
```

### 工作流程

1. 从 `develop` 创建 `feature/xxx` 分支
2. 开发完成后提交并推送到远程
3. 在 GitHub/Gitee 上发起 Pull Request
4. 至少 1 人 Code Review 通过后合并
5. 合并到 `develop` 后定期同步到 `main`

## 五、环境启动清单

### 新成员首次搭建（按顺序执行）

#### 1️⃣ 克隆项目
```bash
git clone <仓库地址> IoT_Smart_Hotel_System
cd IoT_Smart_Hotel_System
npm install -g pnpm   # 或使用 npm/yarn
```

#### 2️⃣ 启动后端依赖（Docker）
```bash
cd docker
docker-compose up -d    # MySQL(3307) + MQTT(1883)
```

#### 3️⃣ 配置后端环境
```bash
cd backend/iot-hotel-backend
cp .env.example .env    # 复制并修改 .env 中的配置
npm install
npx tsc --noEmit        # 类型检查
npm run dev              # 启动后端 :3000
```

#### 4️⃣ 启动前端
```bash
cd frontend/iot-hotel-web
npm install
npm run dev              # 启动前端 :5173
```

#### 5️⃣ 验证
- 浏览器打开 http://localhost:5173
- 默认进入管理端仪表盘
- 侧边栏可切换：管理端 ↔ 前台端 ↔ 客户端

## 七、注意事项

1. **不要提交敏感文件** — `.env`、数据库密码、密钥等已在 `.gitignore` 中排除
2. **不要提交 node_modules 和 dist** — 团队成员自行 `npm install`
3. **不要直接 push 到 main** — 使用 PR 流程确保代码质量
4. **修改公共组件前先沟通** — `stores/`、`api/`、`types/`、`layout/` 是多端共享的
5. **Vue SFC 闭合标签注意** — 用 `</template>` 不是 `</a-template>`（Ant Design 组件标签除外）
6. **前后端接口对接时** — 先确认 `src/types/index.ts` 中的类型定义是否一致
7. **Docker 端口** — MySQL 映射到 `3307`（避免与本机 3306 冲突），MQTT 为 `1883`
